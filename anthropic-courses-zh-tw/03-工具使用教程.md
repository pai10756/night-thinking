## 第一章：什麼是工具使用？

### 學習目標

- 理解什麼是「工具使用」（Tool Use）
- 了解工具使用的應用場景
- 掌握 Claude 使用工具的基本流程

### 工具使用的白話解釋

> **Tool Use（工具使用）**，也稱為 **Function Calling（函式呼叫）**，是一種讓 AI 助手「借用外部工具」來完成任務的能力。

**用一個生活比喻來理解**：想像 Claude 是一位非常聰明的助理，他知道很多事情，但他沒有辦法自己去銀行查你的帳戶餘額。不過，如果你給他一支電話（工具），告訴他銀行的號碼和怎麼查詢，他就能打電話幫你查到餘額，再把結果告訴你。

**關鍵概念**：Claude 本身不會「執行」任何工具。我們告訴 Claude 有哪些工具可以用，Claude 會判斷什麼時候需要用工具，然後「請求」我們去執行，最後我們把結果告訴 Claude。整個過程就像：

1. 你（開發者）告訴 Claude：「你有一個計算機工具可以用」
2. 使用者問 Claude：「1984135 乘以 9343116 等於多少？」
3. Claude 說：「我需要用計算機工具，請幫我算 1984135 × 9343116」
4. 你的程式執行計算，得到結果 18538003464660
5. 你把結果告訴 Claude
6. Claude 回答使用者：「答案是 18538003464660」

### 為什麼工具使用很重要？

- **擴展 Claude 的能力**：Claude 很聰明，但它無法查詢即時股價、無法存取你的資料庫、無法發送電子郵件。透過工具，這些都變得可能。
- **連接你的系統**：可以讓 Claude 與你的資料庫、API（應用程式介面，簡單說就是不同軟體之間溝通的橋樑）、後端服務無縫整合。
- **自動化複雜任務**：把多步驟的工作流程包裝成工具，Claude 可以智慧地在對的時機呼叫對的工具。
- **提升使用者體驗**：使用者用自然語言提問，Claude 在背後呼叫工具取得精確資訊後回覆。
- **彈性擴展**：隨時可以新增或修改工具，讓 Claude 的能力跟著成長。

### 常見應用場景

工具使用的常見類型包括：

- **擷取資訊**：從資料庫、API 或網路服務取得資料（例如天氣、股價、新聞）
- **執行計算**：進行複雜的數學運算、財務分析或統計計算
- **操作資料**：處理、轉換或格式化各種資料
- **與外部系統互動**：發送電子郵件、觸發通知、控制 IoT（物聯網）設備
- **產生內容**：根據使用者需求生成圖表、圖片或格式化文件

更具體的應用案例：

- **企業資料整合**：整合 CRM（客戶關係管理系統）、ERP（企業資源規劃系統）
- **金融分析與報告**：分析財務數據，產生投資報告
- **客戶支援自動化**：連接知識庫和工單系統，提供自動化客服
- **軟體開發輔助**：與程式碼編輯器、版本控制系統整合

### 工具使用的運作流程

> **最重要的一點**：Claude 不會自己執行任何程式碼。我們告訴 Claude 有哪些工具，Claude 告訴我們它想用哪個工具和什麼參數，然後我們自己去執行工具並把結果告訴 Claude。

完整的四個步驟：

**步驟 1：提供工具定義和使用者提示**（我們 → Claude）

- 定義工具的名稱、描述、需要什麼輸入參數
- 附上使用者的問題

**步驟 2：Claude 決定使用工具**（Claude → 我們）

- Claude 判斷是否需要使用工具
- 如果需要，Claude 會回傳一個格式化的「工具使用請求」
- 回應中會標示 `stop_reason` 為 `tool_use`（停止原因：工具使用）

**步驟 3：執行工具並回傳結果**（我們 → Claude）

- 從 Claude 的回應中取出工具名稱和輸入參數
- 在我們這端執行實際的工具功能
- 把結果送回給 Claude

**步驟 4：Claude 根據工具結果回答問題**（Claude → 使用者）

- Claude 收到工具結果後，用這些資訊組合出最終回答

### 假想情境：股價查詢

假設我們要建一個聊天應用程式，讓使用者可以問 Claude 股票價格。

**步驟 0：先寫好工具的功能**

```python
# 這個函式會連接股市 API 查詢即時股價
# def 是 Python 中「定義函式」的關鍵字
# 函式就像一個「小機器」，你給它材料（公司名稱），它給你成品（股價資訊）
def get_stock_price(company):
    # 送出請求到股市 API，查詢某公司的即時股價
    # 回傳包含目前股價資訊的字典（dictionary，一種資料結構）
    pass
```

如果呼叫 `get_stock_price("General Motors")`，可能會得到：

```python
{
  "symbol": "GM",    # 股票代號
  "price": 43.09     # 目前股價
}
```

**步驟 1：定義工具並告訴 Claude**

```python
# 工具定義 —— 用一個「說明書」告訴 Claude 這個工具是什麼、怎麼用
tool_definition = {
    "name": "get_stock_price",                          # 工具名稱
    "description": "查詢指定公司的即時股價",              # 工具描述
    "input_schema": {                                    # 輸入格式說明
        "type": "object",                                # 輸入是一個物件
        "properties": {                                  # 物件的屬性
            "company": {
                "type": "string",                        # 類型是字串（文字）
                "description": "要查詢股價的公司名稱"
            }
        },
        "required": ["company"]                          # 必填欄位
    }
}
```

> **input_schema（輸入格式定義）** 使用一種叫做 **JSON Schema** 的標準格式來描述工具需要什麼輸入。你可以把它想成一份「表格」，告訴 Claude 要填什麼資料、每個欄位是什麼類型。

然後發送請求：

```python
# 向 Claude 發送請求，附上工具定義和使用者的問題
response = client.messages.create(
    model="claude-3-opus-20240229",        # 使用的 Claude 模型
    messages=[{"role": "user", "content": "用 500 美元能買幾股通用汽車？"}],
    max_tokens=500,                         # 最大回應長度
    tools=[tool_definition]                 # 告訴 Claude 可以使用這個工具
)
```

**步驟 2：Claude 回應說要用工具**

```python
{
  "stop_reason": "tool_use",               # Claude 停下來的原因：要用工具
  "tool_use": {
    "name": "get_stock_price",             # 想用的工具名稱
    "input": {
      "company": "General Motors"          # 要查詢的公司
    }
  }
}
```

**步驟 3：我們執行工具，回傳結果**

我們拿到 Claude 指定的公司名稱「General Motors」，呼叫真正的股價查詢函式，得到 `{"symbol": "GM", "price": 43.09}`，然後把結果送回給 Claude。

**步驟 4：Claude 組合最終回答**

> 以目前每股 43.09 美元的價格，你用 500 美元大約可以買 11 股通用汽車的股票。

## 第二章：你的第一個工具

### 學習目標

- 動手實作一個簡單的工具使用範例
- 了解工具定義的格式
- 學會從 Claude 的回應中提取工具資訊

### 為什麼需要計算機工具？

AI 語言模型在處理複雜數學計算時經常會出錯。讓我們來看一個例子：

```python
from anthropic import Anthropic  # 匯入 Anthropic 的 SDK（軟體開發套件）
from dotenv import load_dotenv   # 匯入用來讀取環境變數的工具

load_dotenv()  # 載入環境變數（例如 API 金鑰）

client = Anthropic()  # 建立 Anthropic 客戶端

# 問 Claude 一道數學題
response = client.messages.create(
    model="claude-3-haiku-20240307",   # 使用 Haiku 模型（較快速的版本）
    messages=[{"role": "user", "content": "把 1984135 乘以 9343116，只回答結果"}],
    max_tokens=400
)
print(response.content[0].text)
```

Claude 的回答：`18593367726060`（每次可能不同）

正確答案是：`18538003464660`

差了 `55364261400`！所以我們需要給 Claude 一個計算機工具。

> **比喻**：這就像請一位博學的教授幫你算帳 —— 他什麼知識都有，但算數偶爾會出錯。所以你給他一台計算機（工具），讓他用計算機來處理數學部分。

### 步驟一：寫好工具的功能

```python
# 定義一個簡單的計算機函式
# operation = 運算方式（加、減、乘、除）
# operand1 = 第一個數字
# operand2 = 第二個數字
def calculator(operation, operand1, operand2):
    if operation == "add":          # 加法
        return operand1 + operand2
    elif operation == "subtract":   # 減法
        return operand1 - operand2
    elif operation == "multiply":   # 乘法
        return operand1 * operand2
    elif operation == "divide":     # 除法
        if operand2 == 0:
            raise ValueError("不能除以零。")  # 除以零會報錯
        return operand1 / operand2
    else:
        raise ValueError(f"不支援的運算: {operation}")
```

測試一下：

```python
calculator("add", 10, 3)        # 結果：13
calculator("divide", 200, 25)   # 結果：8.0
```

### 步驟二：定義工具的「說明書」

> 每個工具定義（Tool Definition）包含三個部分：
> - **name**（名稱）：工具的名字
> - **description**（描述）：說明工具做什麼、什麼時候該用
> - **input_schema**（輸入格式）：定義工具需要什麼參數

```python
# 計算機工具的定義 —— 這份「說明書」會送給 Claude 看
calculator_tool = {
    "name": "calculator",  # 工具名稱
    "description": "一個執行基本算術運算的簡單計算機。",  # 工具描述
    "input_schema": {  # 輸入格式定義
        "type": "object",  # 輸入是一個物件
        "properties": {    # 物件包含以下屬性
            "operation": {
                "type": "string",  # 類型：字串
                "enum": ["add", "subtract", "multiply", "divide"],  # 只能是這四個值之一
                "description": "要執行的算術運算。"
            },
            "operand1": {
                "type": "number",  # 類型：數字
                "description": "第一個運算元（第一個數字）。"
            },
            "operand2": {
                "type": "number",
                "description": "第二個運算元（第二個數字）。"
            }
        },
        "required": ["operation", "operand1", "operand2"]  # 三個參數都是必填的
    }
}
```

> **enum（列舉）**：表示這個欄位只能是列出的幾個值之一。就像下拉選單，你只能從「加、減、乘、除」中選一個。

### 步驟三：把工具告訴 Claude 並提問

```python
response = client.messages.create(
    model="claude-3-haiku-20240307",
    messages=[{"role": "user", "content": "把 1984135 乘以 9343116，只回答結果"}],
    max_tokens=300,
    tools=[calculator_tool]  # 告訴 Claude 可以使用計算機工具
)
```

### 步驟四：查看 Claude 的回應

Claude 回傳的不是普通文字，而是一個「工具使用請求」：

```python
response.stop_reason  # 'tool_use' —— Claude 停下來是因為要用工具
```

> **stop_reason（停止原因）**：告訴我們 Claude 為什麼停下來。`tool_use` 表示它想用工具；`end_turn` 表示它說完了。

```python
response.content
# [ToolUseBlock(
#     id='toolu_015wQ7...',           # 這次工具使用的唯一識別碼
#     input={                          # Claude 想傳給工具的參數
#         'operand1': 1984135,
#         'operand2': 9343116,
#         'operation': 'multiply'
#     },
#     name='calculator',               # 想用的工具名稱
#     type='tool_use'
# )]
```

### 步驟五：執行工具並取得結果

```python
# 從 Claude 的回應中取出工具名稱和參數
tool_name = response.content[0].name      # 'calculator'
tool_inputs = response.content[0].input   # {'operand1': 1984135, ...}

# 實際執行計算
operation = tool_inputs["operation"]
operand1 = tool_inputs["operand1"]
operand2 = tool_inputs["operand2"]

result = calculator(operation, operand1, operand2)
print("結果是", result)  # 結果是 18538003464660 ✅ 正確！
```

### 重要提醒：Claude 有時太熱衷於使用工具

如果我們問 Claude「翡翠是什麼顏色？」，它不需要計算機，但有時候 Claude 還是會試圖使用計算機工具。

**解決方法**：加上系統提示（System Prompt），告訴 Claude 只在必要時才用工具。

> **System Prompt（系統提示）**：開發者給 Claude 的「行為準則」，使用者看不到這段指示，但 Claude 會遵守。

```python
response = client.messages.create(
    model="claude-3-haiku-20240307",
    system="你可以使用工具，但只在必要時使用。如果不需要工具，請正常回答。",
    messages=[{"role": "user", "content": "翡翠是什麼顏色？"}],
    max_tokens=400,
    tools=[calculator_tool]
)
# Claude 回答：「翡翠是綠色的。」 —— 沒有使用工具 ✅
```

### 完整範例：把所有步驟組合在一起

```python
def prompt_claude(prompt):
    """
    向 Claude 提問的函式
    會自動判斷 Claude 是否要用工具，並處理相應的流程
    """
    messages = [{"role": "user", "content": prompt}]
    response = client.messages.create(
        model="claude-3-haiku-20240307",
        system="你可以使用工具，但只在必要時使用。如果不需要工具，請正常回答。",
        messages=messages,
        max_tokens=500,
        tools=[calculator_tool],
    )

    if response.stop_reason == "tool_use":
        # Claude 想用工具
        tool_use = response.content[-1]
        tool_name = tool_use.name
        tool_input = tool_use.input

        if tool_name == "calculator":
            print("Claude 想使用計算機工具")
            result = calculator(
                tool_input["operation"],
                tool_input["operand1"],
                tool_input["operand2"]
            )
            print("計算結果：", result)

    elif response.stop_reason == "end_turn":
        # Claude 不需要工具，直接回答
        print("Claude 不需要使用工具")
        print("Claude 的回答：")
        print(response.content[0].text)
```

測試結果：

```python
prompt_claude("我有 23 隻雞，但 2 隻飛走了。還剩幾隻？")
# Claude 想使用計算機工具
# 計算結果：21

prompt_claude("201 乘以 2 是多少？")
# Claude 想使用計算機工具
# 計算結果：402

prompt_claude("幫我寫一首關於海洋的俳句")
# Claude 不需要使用工具
# Claude 的回答：
# 蔚藍汪洋閃，浪花拍打沙灘岸，海洋安詳曲。
```

## 第三章：用工具產生結構化輸出

### 學習目標

- 理解如何利用工具「技巧」來強制產生結構化回應
- 學會用這個方法讓 Claude 產生格式化的 JSON 資料

### 核心概念

> **JSON（JavaScript Object Notation）**：一種通用的資料格式，用來組織和傳輸資料。就像一份填好的表格，有固定的欄位名稱和對應的值。例如：`{"name": "小明", "age": 25}`

**這個章節的核心技巧**：每當 Claude 想使用工具時，它會自動按照我們定義的格式來回應。我們可以利用這一點 —— 定義一個「假工具」，不是真的要用它，只是要讓 Claude 按照我們想要的格式回應。

> **比喻**：這就像你給助理一份空白表格，告訴他「用這個格式回報」。助理以為他在填表格（使用工具），但其實你只是想要一份格式整齊的報告。

在前一章中，當 Claude 想用計算機工具時，它回應了這樣的格式：

```python
{
    'operand1': 1984135,
    'operand2': 9343116,
    'operation': 'multiply'
}
```

這看起來就是結構化的 JSON！我們可以利用這個特性。

### 範例一：情感分析

假設我們想讓 Claude 分析一段文字的情緒，並回傳這樣格式的資料：

```json
{
  "negative_score": 0.6,
  "neutral_score": 0.3,
  "positive_score": 0.1
}
```

我們只需要定義一個「工具」來描述這個格式：

```python
# 定義一個「假工具」—— 我們不會真的執行它
# 目的只是讓 Claude 按照這個格式回應
tools = [
    {
        "name": "print_sentiment_scores",
        "description": "輸出指定文字的情感分數。",
        "input_schema": {
            "type": "object",
            "properties": {
                "positive_score": {
                    "type": "number",
                    "description": "正面情感分數，範圍 0.0 到 1.0。"
                },
                "negative_score": {
                    "type": "number",
                    "description": "負面情感分數，範圍 0.0 到 1.0。"
                },
                "neutral_score": {
                    "type": "number",
                    "description": "中性情感分數，範圍 0.0 到 1.0。"
                }
            },
            "required": ["positive_score", "negative_score", "neutral_score"]
        }
    }
]
```

然後向 Claude 發送請求：

```python
tweet = "我超級討厭醃黃瓜。我真的非常厭惡醃黃瓜。它們就是垃圾。"

query = f"""
<text>
{tweet}
</text>

只使用 print_sentiment_scores 工具。
"""

response = client.messages.create(
    model="claude-3-sonnet-20240229",
    max_tokens=4096,
    tools=tools,
```

```python
def answer_question(question):
    """
    向 Claude 提問，如果需要會自動搜尋維基百科
    """
    # 系統提示：告訴 Claude 只在必要時使用工具
    system_prompt = """
    使用者會問你問題。
    如果回答問題需要你訓練資料中沒有的資訊，可以使用 get_article 工具來取得最新的維基百科文章。
    如果你可以不需要額外資訊就回答問題，請直接回答。
    只在需要時才呼叫工具。
    """

    # 進一步優化提示，讓回答更簡潔
    prompt = f"""
    回答以下問題 <question>{question}</question>
    當你能回答問題時，盡量簡短回答，並用 <answer> 標籤包住答案。
    """

    messages = [{"role": "user", "content": prompt}]

    response = client.messages.create(
        model="claude-3-sonnet-20240229",
        system=system_prompt,
        messages=messages,
        max_tokens=1000,
        tools=[article_search_tool]
    )

    if response.stop_reason == "tool_use":
        # Claude 想用工具
        tool_use = response.content[-1]
        tool_name = tool_use.name
        tool_input = tool_use.input
        messages.append({"role": "assistant", "content": response.content})

        if tool_name == "get_article":
            search_term = tool_input["search_term"]
            print(f"Claude 想查 {search_term} 的文章")
            wiki_result = get_article(search_term)

            # 回傳工具結果
            tool_response = {
                "role": "user",
                "content": [{
                    "type": "tool_result",
                    "tool_use_id": tool_use.id,
                    "content": wiki_result
                }]
            }
            messages.append(tool_response)

            # 讓 Claude 根據結果回答
            response = client.messages.create(
                model="claude-3-sonnet-20240229",
                system=system_prompt,
                messages=messages,
                max_tokens=1000,
                tools=[article_search_tool]
            )
            print("Claude 的最終答案：")
            print(response.content[0].text)
    else:
        # Claude 不需要工具
        print("Claude 不需要呼叫工具")
        print(response.content[0].text)
```

## 第五章：工具選擇控制

### 學習目標

- 理解 `tool_choice` 參數的三種模式
- 學會在不同場景中選擇正確的模式

### tool_choice 是什麼？

> **tool_choice（工具選擇）**：一個參數，讓你控制 Claude 如何決定是否使用工具。有三種模式：

| 模式 | 說明 | 白話解釋 |
|------|------|----------|
| `auto` | Claude 自己決定要不要用工具 | 「你看著辦」 |
| `any` | Claude 必須用一個工具，但可以選哪個 | 「你一定要用工具，但選哪個隨你」 |
| `tool` | Claude 必須用指定的特定工具 | 「你一定要用這個工具」 |

### 模式一：Auto（自動）

這是預設模式。Claude 會根據情況自行判斷是否需要使用工具。

```python
# 設定為 auto 模式
tool_choice = {"type": "auto"}
```

**範例**：給 Claude 一個網路搜尋工具

### 學習目標

- 處理多步驟的工具呼叫
- 學會優化系統提示來改善聊天體驗

### 專案介紹

我們要建一個簡單的客服聊天機器人，服務一家虛構的電子產品公司「TechNova」。機器人能使用以下工具：

| 工具名稱 | 功能 | 白話說明 |
|----------|------|----------|
| `get_user` | 透過 email、使用者名稱或電話查詢使用者 | 「幫我找到這個客戶」 |
| `get_order_by_id` | 用訂單編號查詢訂單 | 「幫我查這張訂單」 |
| `get_customer_orders` | 查詢某客戶的所有訂單 | 「幫我列出這個人的所有訂單」 |
| `cancel_order` | 取消訂單 | 「幫我取消這張訂單」 |

### 建立模擬資料庫

> **資料庫（Database）**：儲存和管理資料的系統。這裡我們用一個 Python 類別來模擬真實的資料庫。

```python
class FakeDatabase:
    """模擬的客戶與訂單資料庫"""
    def __init__(self):
        # 客戶資料
        self.customers = [
            {"id": "1213210", "name": "John Doe", "email": "john@gmail.com",
             "phone": "123-456-7890", "username": "johndoe"},
            {"id": "2837622", "name": "Priya Patel", "email": "priya@candy.com",
             "phone": "987-654-3210", "username": "priya123"},
            # ... 更多客戶
        ]

        # 訂單資料
        self.orders = [
            {"id": "24601", "customer_id": "1213210",
             "product": "Wireless Headphones", "quantity": 1,
             "price": 79.99, "status": "Shipped"},      # 已出貨
            {"id": "13579", "customer_id": "1213210",
             "product": "Smartphone Case", "quantity": 2,
             "price": 19.99, "status": "Processing"},    # 處理中
            # ... 更多訂單
        ]

    def get_user(self, key, value):
        """用 email、使用者名稱或電話查找客戶"""
        for customer in self.customers:
            if customer[key] == value:
                return customer
        return f"找不到 {key} 為 {value} 的客戶"

    def get_order_by_id(self, order_id):
        """用訂單編號查詢訂單"""
        for order in self.orders:
            if order["id"] == order_id:
                return order
        return "找不到此訂單"

    def get_customer_orders(self, customer_id):
        """列出某客戶的所有訂單"""
        return [order for order in self.orders
                if order["customer_id"] == customer_id]

    def cancel_order(self, order_id):
        """取消訂單（只有「處理中」的訂單可以取消）"""
        order = self.get_order_by_id(order_id)
        if order["status"] == "Processing":
            order["status"] = "Cancelled"
            return "已取消該訂單"
        else:
            return f"無法取消訂單 {order_id}（狀態：{order['status']}）"
```

### 定義四個工具


### 處理工具呼叫的函式

```python
def process_tool_call(tool_name, tool_input):
    """
    根據 Claude 指定的工具名稱和參數，執行對應的資料庫操作
    這是一個「翻譯器」—— 把 Claude 的請求轉換成實際的資料庫查詢
    """
    if tool_name == "get_user":
        return db.get_user(tool_input["key"], tool_input["value"])
    elif tool_name == "get_order_by_id":
        return db.get_order_by_id(tool_input["order_id"])
    elif tool_name == "get_customer_orders":
        return db.get_customer_orders(tool_input["customer_id"])
    elif tool_name == "cancel_order":
        return db.cancel_order(tool_input["order_id"])
```

### Claude 會自己選擇正確的工具

當使用者說「幫我查訂單，我的 email 是 john@gmail.com」時：

1. Claude 先呼叫 `get_user` 工具（用 email 查客戶）→ 找到 John Doe，ID 是 1213210
2. 我們把結果告訴 Claude
3. Claude 再呼叫 `get_customer_orders` 工具（用客戶 ID 查所有訂單）→ 找到 3 張訂單
4. 我們把結果告訴 Claude
5. Claude 組合最終回答告訴使用者他的訂單詳情

> **注意**：Claude 自動判斷了需要「先查人、再查訂單」的兩步驟流程！這就是多工具使用的威力。

### 建立互動式聊天腳本

```python
import re

def extract_reply(text):
    """從 Claude 的回應中擷取 <reply> 標籤內的內容"""
    pattern = r'<reply>(.*?)</reply>'
    match = re.search(pattern, text, re.DOTALL)
    if match:
        return match.group(1)
    return None

def simple_chat():
    """互動式聊天機器人主函式"""
    system_prompt = """
    你是線上零售商 TechNova 的客服聊天機器人。
    你的工作是幫助使用者查詢帳號、訂單以及取消訂單。
    回答要簡短有幫助。
    你可以使用一組工具，但只在需要時使用。
    如果你沒有足夠的資訊來正確使用工具，請向使用者詢問後續問題以取得必要的資訊。
    在沒有從使用者那裡取得必要資料之前，不要呼叫任何工具。

    在每次對話中，你會先思考你的回應。
    思考完畢後，撰寫面向使用者的回應。
    請務必將所有面向使用者的回應放在 <reply></reply> XML 標籤中，以便於解析。
    """

    user_message = input("\n使用者：")
    messages = [{"role": "user", "content": user_message}]

    while True:
        # 如果最後一則訊息是助手的，取得使用者的下一則輸入
        if messages[-1].get("role") == "assistant":
            user_message = input("\n使用者：")
            messages.append({"role": "user", "content": user_message})

        # 向 Claude 發送請求
        response = client.messages.create(
            model=MODEL_NAME,
            system=system_prompt,
            max_tokens=4096,
            tools=tools,
            messages=messages
        )

        # 把 Claude 的回應加到訊息列表
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason == "tool_use":
            # Claude 想用工具
            tool_use = response.content[-1]
            tool_name = tool_use.name
            tool_input = tool_use.input
            print(f"======Claude 想使用 {tool_name} 工具======")

            # 執行工具
            tool_result = process_tool_call(tool_name, tool_input)

            # 把工具結果加到訊息列表
            messages.append({
                "role": "user",
                "content": [{
                    "type": "tool_result",
                    "tool_use_id": tool_use.id,
                    "content": str(tool_result),
                }],
            })
        else:
            # Claude 不需要工具，直接回答使用者
            model_reply = extract_reply(response.content[0].text)
            print("\nTechNova 客服：" + f"{model_reply}")
```

### 系統提示的重要性

在開發過程中，我們發現 Claude 有時會出現不理想的行為：

**問題 1：Claude 會告訴使用者它在用什麼工具**

解法：加上系統提示告訴 Claude 它是 TechNova 的客服，並且不要提及工具。

**問題 2：Claude 在沒有足夠資訊時就亂呼叫工具**

例如使用者說「幫我取消訂單」但沒提供任何身份資訊，Claude 就自己編了一個 email 去查詢。

解法：在系統提示中加上：「如果你沒有足夠的資訊來正確使用工具，請向使用者詢問後續問題」和「在沒有從使用者那裡取得必要資料之前，不要呼叫任何工具」。

**問題 3：Claude（尤其是 Opus 模型）會在回應中顯示思考過程**

Claude 有時會輸出 `<thinking>` 或 `<reflection>` 標籤的內部思考過程，這對使用者來說體驗很差。

解法：要求 Claude 把面向使用者的回應放在 `<reply></reply>` 標籤中，然後在程式碼中只擷取標籤內的內容。

### 完成後的注意事項

> 這是一個教學示範，**不適合直接用於正式產品**。在真實世界中，你還需要：
> - 加入身份驗證（不能讓任何知道別人 email 的人就能取消訂單）
> - 提供完整的公司知識庫讓 Claude 參考
> - 進行全面的測試確保機器人在各種情況下表現良好
> - 加入錯誤處理和輸入驗證

## 總結

### 核心概念回顧

| 概念 | 白話說明 |
|------|----------|
| **Tool Use（工具使用）** | 讓 AI 助手「借用」外部功能來完成任務 |
| **Tool Definition（工具定義）** | 告訴 AI 工具是什麼、怎麼用的「說明書」 |
| **input_schema（輸入格式）** | 定義工具需要什麼參數的「表格格式」 |
| **tool_result（工具結果）** | 把工具執行結果送回給 AI 的特殊訊息格式 |
| **tool_choice（工具選擇）** | 控制 AI 如何決定是否使用工具的設定 |
| **stop_reason（停止原因）** | 告訴我們 AI 為什麼停下來（要用工具、還是說完了） |
| **JSON Schema** | 一種標準格式，用來描述資料應該長什麼樣子 |
| **System Prompt（系統提示）** | 給 AI 的行為準則，使用者看不到但 AI 會遵守 |
| **API（應用程式介面）** | 不同軟體之間溝通的橋樑 |

### 四步驟流程速查

```
1. [你] 定義工具 + 發送使用者問題 → Claude
2. [Claude] 判斷要用什麼工具 → 你
3. [你] 執行工具 + 回傳結果 → Claude
4. [Claude] 根據結果回答使用者 → 使用者
```

### 三種工具選擇模式速查

```
auto  → Claude 自己決定（預設模式，最常用）
any   → 一定要用工具，但選哪個隨 Claude（適合只能透過工具溝通的場景）
tool  → 一定要用指定的工具（適合強制產生特定格式的輸出）
```
