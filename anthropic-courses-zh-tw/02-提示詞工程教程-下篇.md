# 第五章：格式化輸出與「替 Claude 說話」

> **本章重點：** 你可以要求 Claude 用特定格式回答，也可以「替它開頭」來引導它的回答方向。就像填空題一樣——你先寫好開頭幾個字，Claude 就會順著往下接。

## 用 XML 標籤整理輸出

> **XML 標籤（XML Tags）**：一種用角括號 `< >` 包起來的標記，像是書的章節標題，幫助你把內容分門別類。例如 `<haiku>` 和 `</haiku>` 就像一對書擋，把俳句夾在中間。

Claude 可以用各式各樣的格式來輸出內容，你只需要告訴它就好。

其中一種方式是使用 XML 標籤，把回答和多餘的文字分開。你在前面章節已經學過可以用 XML 標籤讓提示詞更清楚，其實你也可以請 Claude **用 XML 標籤讓它的輸出更容易理解**。

### 範例：用標籤包住詩句

還記得第二章的「詩句前言問題」嗎？我們當時請 Claude 跳過前言直接寫詩。其實我們也可以**叫 Claude 把詩放在 XML 標籤裡**。

```python
# 變數內容
ANIMAL = "Rabbit"  # 動物名稱

# 提示詞模板
PROMPT = f"Please write a haiku about {ANIMAL}. Put it in <haiku> tags."
# 翻譯：「請寫一首關於{動物}的俳句，放在 <haiku> 標籤裡。」

# 送出並印出 Claude 的回覆
print(get_completion(PROMPT))
```

> **為什麼要這樣做？** 把輸出放在 XML 標籤裡，就像把信放進信封——你（或你的程式）可以輕鬆地只取出標籤之間的內容，忽略其他多餘的話。這在需要用程式自動處理回覆時特別有用。

### 「替 Claude 說話」——預填回覆

> **預填回覆（Prefill）**：在 Claude 的「助手回合」中預先放入一些文字，讓 Claude 以為自己已經說了這些話，然後從這裡接著說下去。這就像在考試時，題目已經幫你寫好了開頭「答：根據...」，你只需要接著寫下去。

這個技巧叫做「替 Claude 說話」（speaking for Claude）或「預填 Claude 的回覆」（prefilling Claude's response）。

```python
# 變數內容
ANIMAL = "Cat"  # 動物：貓

# 提示詞
PROMPT = f"Please write a haiku about {ANIMAL}. Put it in <haiku> tags."
# 翻譯：「請寫一首關於貓的俳句，放在 <haiku> 標籤裡。」

# 預填文字——我們先「替 Claude 開頭」
PREFILL = "<haiku>"

# Claude 會直接從 <haiku> 後面開始寫，不會再加多餘的開場白
print(get_completion(PROMPT, prefill=PREFILL))
```

> **白話比喻：** 想像你請朋友唱一首歌，但你先哼了前兩個音。朋友自然就會接著你的旋律唱下去，而不是從頭自我介紹再開始唱。預填就是這個道理。

### 強制輸出 JSON 格式

> **JSON（JavaScript Object Notation）**：一種常見的資料格式，像是一張有標籤的清單。例如：`{"姓名": "小明", "年齡": 25}`。程式特別喜歡這種格式，因為很容易讀取。

如果你想讓 Claude 輸出 JSON 格式，可以在預填中放一個左大括號 `{`，Claude 就會順著寫出完整的 JSON。

```python
ANIMAL = "Cat"

# 提示詞：請寫俳句，用 JSON 格式，包含三行
PROMPT = f'Please write a haiku about {ANIMAL}. Use JSON format with the keys as "first_line", "second_line", and "third_line".'

# 預填一個左大括號，強制 Claude 輸出 JSON
PREFILL = "{"

print(get_completion(PROMPT, prefill=PREFILL))
# 結果會像：{ "first_line": "...", "second_line": "...", "third_line": "..." }
```

### 多變數 + XML 標籤的組合運用

你也可以同時使用多個變數和 XML 標籤：

```python
# 第一個變數：一封 email
EMAIL = "Hi Zack, just pinging you for a quick update on that prompt you were supposed to write."

# 第二個變數：風格
ADJECTIVE = "olde english"  # 古英文風格

# 提示詞：請把這封 email 改成古英文風格
PROMPT = f"Hey Claude. Here is an email: <email>{EMAIL}</email>. Make this email more {ADJECTIVE}. Write the new version in <{ADJECTIVE}_email> XML tags."

# 預填開頭標籤
PREFILL = f"<{ADJECTIVE}_email>"

print(get_completion(PROMPT, prefill=PREFILL))
```

### 進階技巧：用停止序列節省成本

> **停止序列（Stop Sequences）**：告訴 Claude「看到這個標記就停下來別再說了」。這可以省錢（因為 API 按字數收費），也能讓回覆更精確。

如果你透過 API 呼叫 Claude，可以把結束的 XML 標籤（例如 `</haiku>`）設為停止序列，這樣 Claude 寫完詩就會停下來，不會再加多餘的結語。

# 第六章：預先思考（讓 Claude 一步步想）

> **本章重點：** 就像考試時，先在草稿紙上列出思路再寫答案，正確率會更高。Claude 也一樣——讓它先「想一想」再回答，準確度會大幅提升。

## 為什麼要讓 Claude「想出來」？

> **逐步思考（Thinking Step by Step）**：請 Claude 把思考過程寫出來，而不是直接給答案。這就像數學老師要求你「寫出計算過程」——過程寫出來了，答案自然更容易正確。

如果有人把你叫醒後馬上問你好幾個複雜問題，你大概答不好。但如果給你時間先想一想呢？肯定會答得更好。

Claude 也是一樣。

**重要原則：思考必須「出聲」才算數。** 你不能叫 Claude 「先想再只給答案」——如果思考過程沒有被寫出來，那就等於沒有思考。

> **白話比喻：** 這就像叫一個學生「心算」和「在紙上列式計算」的差別。心算可能算錯，但列在紙上一步步算，就不容易出錯。Claude 的「紙上列式」就是把思考過程寫出來。

### 範例：電影評論的情感判斷

下面這則評論帶有諷刺意味：「這部電影讓我大開眼界。順帶一提（完全無關），我從 1900 年就住在石頭下面了。」

人類讀者可以看出這是在諷刺——說「完全無關」其實就是在暗示有關。但 Claude 可能會太字面地理解「無關」這個詞。

```python
# 直接問——Claude 可能判斷錯誤
PROMPT = """Is this movie review sentiment positive or negative?

This movie blew my mind with its freshness and originality. In totally unrelated news, I have been living under a rock since the year 1900."""
# 翻譯：「這篇電影評論是正面還是負面的？
# 這部電影的新穎和原創讓我大開眼界。順帶一提，我從 1900 年起就住在石頭底下。」
```

改良版——讓 Claude 先列出正反論點再回答：

```python
# 系統提示：給 Claude 一個角色
SYSTEM_PROMPT = "You are a savvy reader of movie reviews."
# 翻譯：「你是一個很有見地的電影評論讀者。」

# 提示詞：先列正反論點，再回答
PROMPT = """Is this review sentiment positive or negative? First, write the best arguments for each side in <positive-argument> and <negative-argument> XML tags, then answer.

This movie blew my mind with its freshness and originality. In totally unrelated news, I have been living under a rock since 1900."""
# 翻譯：「這篇評論是正面還是負面？先在 XML 標籤中列出正反兩方的最佳論點，然後再回答。」
```

> 透過先列出正反論點，Claude 更有可能注意到「我從 1900 年起就住在石頭下面」是一句諷刺，暗示這個人根本沒看過其他電影，所以說「大開眼界」不是真正的讚美。

### Claude 對順序的敏感性

> **順序效應（Order Sensitivity）**：Claude 傾向於選擇列表中**後面**的選項。這可能是因為訓練資料中，後面的選項更常是正確答案。

在大多數情況下（但不是全部），Claude 更傾向於選擇兩個選項中的**第二個**。所以如果你把「正面論點」和「負面論點」的順序互換，可能會影響 Claude 的最終判斷。

> **實用建議：** 如果你發現 Claude 總是偏向某個選項，試著調換選項順序看看效果。

### 範例：需要思考的事實題

```python
# 直接問——Claude 可能給出錯誤答案（演員出生年份不對）
PROMPT = "Name a famous movie starring an actor who was born in the year 1956."
# 翻譯：「說出一部由 1956 年出生的演員主演的著名電影。」
```

改良版——先讓 Claude 在「腦力激盪」標籤中思考：

```python
# 讓 Claude 先列出演員和出生年份，再給答案
PROMPT = "Name a famous movie starring an actor who was born in the year 1956. First brainstorm about some actors and their birth years in <brainstorm> tags, then give your answer."
# 翻譯：「...先在 <brainstorm> 標籤中腦力激盪一些演員和他們的出生年份，然後再給答案。」
```

> **白話比喻：** 如果有人問你「說出一個 1985 年出生的名人」，你直接猜可能會猜錯。但如果先在腦中列出幾個名人和他們的出生年份，再從中挑選，就容易答對了。

# 第七章：用範例引導（少量示範提示法）

> **本章重點：** 「給 Claude 看幾個正確的範例」是最有效的引導方式之一。就像教小孩寫字，與其解釋筆畫規則，不如直接寫給他看。

## 什麼是少量示範提示法？

> **少量示範提示法（Few-Shot Prompting）**：在提示詞中提供幾個「問答範例」，讓 Claude 從範例中學會你想要的回答風格和格式。
>
> - **零次提示（Zero-Shot）**：不給任何範例，直接問
> - **一次提示（One-Shot）**：給一個範例
> - **少量提示（Few-Shot）**：給幾個範例
>
> 範例越多，Claude 的表現通常越好。

**給 Claude 看你想要的行為範例（或你不想要的行為範例），在以下方面極其有效：**
- 讓 Claude 給出正確答案
- 讓 Claude 用正確的格式回答

### 範例：打造「家長機器人」

假設你是開發者，要建一個回答小朋友問題的「家長機器人」。Claude 預設的回答風格非常正式、冷冰冰的，這會讓小朋友失望。

```python
# 直接問——Claude 的回答太正式了
PROMPT = "Will Santa bring me presents on Christmas?"
# 翻譯：「聖誕老人會送我禮物嗎？」

# Claude 可能會長篇大論地解釋聖誕老人的文化背景...
```

與其花時間描述你想要的語氣，不如**直接給幾個理想回答的範例**：

```python
# 用範例教 Claude 回答的語氣
PROMPT = """Please complete the conversation by writing the next line, speaking as "A".
Q: Is the tooth fairy real?
A: Of course, sweetie. Wrap up your tooth and put it under your pillow tonight. There might be something waiting for you in the morning.
Q: Will Santa bring me presents on Christmas?"""
# 翻譯：「請以 A 的身份完成對話。
# 問：牙仙是真的嗎？
# 答：當然啦，寶貝。今晚把牙齒包好放在枕頭下面，明天早上可能會有驚喜喔。
# 問：聖誕老人會送我禮物嗎？」

# Claude 會模仿上面溫暖的語氣回答！
```

> **白話比喻：** 範例就像是「照著做」的食譜。你給 Claude 看一道完成的菜，它就知道下一道菜該怎麼做了。

### 範例：用範例教格式

你也可以用範例教 Claude 特定的輸出格式。以下範例教 Claude 從文章中提取人名和職業：

```python
PROMPT = """Silvermist Hollow, a charming village, was home to an extraordinary group of individuals...
（一段描述人物的文章）
<individuals>
1. Dr. Liam Patel [NEUROSURGEON]
2. Olivia Chen [ARCHITECT]
3. Ethan Kovacs [MUSICIAN AND COMPOSER]
4. Isabella Torres [CHEF]
</individuals>

（另一段描述人物的文章）
<individuals>
1. Oliver Hamilton [CHEF]
2. Elizabeth Chen [LIBRARIAN]
3. Isabella Torres [ARTIST]
4. Marcus Jenkins [COACH]
</individuals>

（第三段文章——這次讓 Claude 自己提取）
Oak Valley, a charming small town..."""

# 預填開頭標籤
PREFILL = "<individuals>"

# Claude 會模仿前兩個範例的格式，自動提取人名和職業！
```

> **重點：** 你不需要一步步解釋「提取名字，然後提取職業，然後用方括號...」這些複雜規則。只要給兩個正確的範例，Claude 就能自己推斷出格式規則。

# 第八章：避免幻覺

> **本章重點：** AI 有時會編造看似合理但其實不正確的資訊，這叫做「幻覺」。好消息是，有很多技巧可以大幅減少這種情況。

## 什麼是幻覺？

> **幻覺（Hallucination）**：AI 產生的回答中包含了不真實或沒有根據的資訊。就像一個想討好你的朋友，不知道答案卻硬要回答，結果說了不正確的話。Claude 有時候為了「盡力幫忙」，會編造出看似合理的答案。

**壞消息：** Claude 有時候會「幻覺」，產生不真實或缺乏根據的說法。

**好消息：** 有很多方法可以減少幻覺，包括：
- 給 Claude 一個「不知道就說不知道」的出口
- 要求 Claude 先找到證據再回答

### 技巧一：給 Claude 一個「出口」

> **給出口（Giving Claude an Out）**：明確告訴 Claude「如果你不確定，可以說不知道」。這就像告訴學生「不會寫的題目可以跳過，不要亂猜」。

```python
# 不給出口——Claude 會編造答案
PROMPT = "Who is the heaviest hippo of all time?"
# 翻譯：「史上最重的河馬是誰？」
# Claude 可能會編造幾隻「有名的大河馬」...

# 給出口——Claude 會承認不確定
PROMPT = "Who is the heaviest hippo of all time? Only answer if you know the answer with certainty."
# 翻譯：「史上最重的河馬是誰？只有在你確定知道答案時才回答。」
# Claude 會誠實地說它不確定這個資訊。
```

> **白話比喻：** 就像在餐廳點菜時問服務生「你們的招牌菜是什麼？」如果服務生不知道，有兩種反應：一種是隨便推薦一道（幻覺），另一種是說「讓我幫您問一下廚師」（誠實）。給 Claude 出口，就是鼓勵它選擇第二種反應。

### 技巧二：先找證據再回答

> **引用為先（Evidence First）**：要求 Claude 先從文件中找出相關的引用段落，然後基於這些引用來回答。這就像寫論文要附上參考文獻——有根據的答案比較不容易出錯。

當 Claude 處理長文件時，可能會被文件中「看起來相關但其實不完全對」的資訊誤導。

**範例情境：** 給 Claude 一份 Matterport 公司的 SEC 年報，然後問「2020 年 5 月 31 日 Matterport 有多少訂閱用戶？」

```python
# 不好的做法——Claude 可能會被文件中相近但不精確的數字誤導
PROMPT = """<question>What was Matterport's subscriber base on the precise date of May 31, 2020?</question>
Please read the below document. Then write a brief numerical answer inside <answer> tags.
<document>（很長的文件...）</document>"""

# 好的做法——先要求 Claude 提取相關引文，再判斷
PROMPT = """<question>What was Matterport's subscriber base on the precise date of May 31, 2020?</question>
Please read the below document. Then, in <scratchpad> tags, pull the most relevant quote from the document and consider whether it answers the user's question or whether it lacks sufficient detail. Then write a brief numerical answer in <answer> tags.
<document>（很長的文件...）</document>"""
# 翻譯：「...在 <scratchpad>（草稿區）標籤中，找出文件中最相關的引文，
# 並思考它是否真的回答了使用者的問題、還是缺乏足夠的細節。」
```

> **<scratchpad> 標籤**就像 Claude 的「草稿紙」。讓它先在草稿紙上找出證據並思考，再給出正式答案。這樣一來，Claude 更容易發現「文件裡其實沒有 2020 年 5 月 31 日的精確數字」，而不是硬拼湊一個答案。

### 進階技巧：降低溫度參數

> **溫度（Temperature）**：控制 Claude 回答「創意程度」的參數，範圍 0 到 1。
> - **溫度 0**：回答最穩定、最一致，像是照本宣科的播報員
> - **溫度 1**：回答最有創意、最多變化，像是即興演出的喜劇演員
>
> 降低溫度可以減少幻覺，因為 Claude 不會做太多「創意發揮」。

## 練習題

### 練習 8.1 — Beyonce 幻覺

Claude 可能會錯誤地說 Renaissance 是 Beyonce 的「第八張」錄音室專輯（其實是第七張）。修改提示詞，給 Claude 一個出口，讓它在不確定時不要亂回答。

### 練習 8.2 — 財報幻覺

給 Claude 一份 Matterport 的財報，問「從 2018 年 12 月到 2022 年 12 月，Matterport 的訂閱用戶增長了多少倍？」（正確答案是 49 倍。）透過要求引用來讓 Claude 更準確地回答。

# 第九章：從零打造複雜提示詞

> **本章重點：** 把前面學的所有技巧組合在一起，用一套系統化的「提示詞模板」來處理複雜的真實世界任務。這就像做菜——前面學的是切菜、調味、火候等個別技巧，現在要把它們組合起來做一道完整的大菜。

## 複雜提示詞的建議結構

恭喜你來到最後一章！現在是把所有東西整合在一起的時候了。

以下是一套**建議的複雜提示詞結構**。並非所有提示詞都需要每一個元素，你可以自由增減。一般建議：**先把所有元素都用上讓提示詞先跑起來，然後再精簡優化。**

### 十大元素一覽

| 元素 | 說明 | 白話比喻 |
|------|------|----------|
| 1. 使用者角色 | API 呼叫的 `user` 角色 | 誰在問問題 |
| 2. 任務背景 | Claude 的角色定位和總體目標 | 「你是一位職涯顧問」 |
| 3. 語氣設定 | 回答的風格和語氣 | 「請用友善的語氣」 |
| 4. 詳細任務描述與規則 | 具體要做什麼、有哪些限制 | 遊戲規則說明 |
| 5. 範例 | 理想回答的範例 | 「照著這個範例做」 |
| 6. 輸入資料 | 需要處理的資料 | 考試的閱讀材料 |
| 7. 即時任務提醒 | 在最後再次提醒要做什麼 | 「現在請回答這個問題」 |
| 8. 逐步思考 | 要求先思考再回答 | 「先在草稿紙上想想」 |
| 9. 輸出格式 | 指定回答的格式 | 「請用列表回答」 |
| 10. 預填回覆 | 替 Claude 開個頭 | 「答：」 |

> **順序大致有講究：** 任務背景放前面，輸出格式放後面。但有些元素的順序可以彈性調整。如果你按照上面的順序來，通常就是一個不錯的起點。

### 範例一：職涯顧問聊天機器人

以下是一個完整的複雜提示詞，讓 Claude 扮演一位叫「Joe」的 AI 職涯顧問：

```python
# ============= 輸入變數 =============

# 對話歷史（之前的對話紀錄）
HISTORY = """Customer: Give me two possible careers for sociology majors.
Joe: Here are two potential careers for sociology majors:
- Social worker...
- Human resources specialist..."""

# 使用者的新問題
QUESTION = "Which of the two careers requires more than a Bachelor's degree?"
# 翻譯：「這兩個職業中，哪個需要學士以上學位？」

# ============= 提示詞元素 =============

# 元素 2：任務背景
TASK_CONTEXT = "You will be acting as an AI career coach named Joe created by the company AdAstra Careers. Your goal is to give career advice to users."
# 翻譯：「你是一位叫 Joe 的 AI 職涯顧問，由 AdAstra Careers 公司創建。你的目標是提供職涯建議。」

# 元素 3：語氣設定
TONE_CONTEXT = "You should maintain a friendly customer service tone."
# 翻譯：「你應該保持友善的客服語氣。」

# 元素 4：詳細任務描述與規則
TASK_DESCRIPTION = """Here are some important rules for the interaction:
- Always stay in character, as Joe, an AI from AdAstra Careers
- If you are unsure how to respond, say "Sorry, I didn't understand that. Could you rephrase your question?"
- If someone asks something irrelevant, say, "Sorry, I am Joe and I give career advice. Do you have a career question today I can help you with?" """
# 翻譯：「互動的重要規則：
# - 永遠以 Joe 的身份回答
# - 不確定時說「抱歉，我沒聽懂，能換個方式問嗎？」
# - 遇到無關問題說「抱歉，我是 Joe，我提供職涯建議。你有職涯問題嗎？」」

# 元素 5：範例
EXAMPLES = """Here is an example of how to respond in a standard interaction:
<example>
Customer: Hi, how were you created and what do you do?
Joe: Hello! My name is Joe, and I was created by AdAstra Careers to give career advice. What can I help you with today?
</example>"""
# 翻譯：「以下是標準互動的回答範例...」

# 元素 6：輸入資料（對話紀錄 + 問題）
INPUT_DATA = f"""Here is the conversational history (between the user and you) prior to the question:
<history>{HISTORY}</history>
Here is the user's question:
<question>{QUESTION}</question>"""
# 翻譯：「以下是之前的對話紀錄...以下是使用者的問題...」

# 元素 7：即時任務提醒
IMMEDIATE_TASK = "How do you respond to the user's question?"
# 翻譯：「你要怎麼回答使用者的問題？」

# 元素 8：逐步思考
PRECOGNITION = "Think about your answer first before you respond."
# 翻譯：「回答前先想一想。」

# 元素 9：輸出格式
OUTPUT_FORMATTING = "Put your response in <response></response> tags."
# 翻譯：「把回答放在 <response> 標籤裡。」

# 元素 10：預填回覆
PREFILL = "[Joe] <response>"

# ============= 組合所有元素 =============
# 把上面所有元素按順序串起來，組成完整的提示詞
PROMPT = ""
if TASK_CONTEXT: PROMPT += f"{TASK_CONTEXT}"
if TONE_CONTEXT: PROMPT += f"\n\n{TONE_CONTEXT}"
if TASK_DESCRIPTION: PROMPT += f"\n\n{TASK_DESCRIPTION}"
if EXAMPLES: PROMPT += f"\n\n{EXAMPLES}"
if INPUT_DATA: PROMPT += f"\n\n{INPUT_DATA}"
if IMMEDIATE_TASK: PROMPT += f"\n\n{IMMEDIATE_TASK}"
if PRECOGNITION: PROMPT += f"\n\n{PRECOGNITION}"
if OUTPUT_FORMATTING: PROMPT += f"\n\n{OUTPUT_FORMATTING}"
```

> **白話比喻：** 這就像寫一封非常詳細的工作指示信。你告訴新員工：「你的角色是...，你的語氣應該是...，遇到這種情況要...，參考這個範例...，現在有個客戶問了這個問題...，先想想再回答...，請用這個格式寫。」

### 範例二：法律服務助手

法律領域的提示詞通常更複雜，因為需要：
- 解讀長文件
- 處理複雜主題
- 以非常特定的格式輸出
- 遵循多步驟的分析流程

```python
# 任務背景
TASK_CONTEXT = "You are an expert lawyer."
# 翻譯：「你是一位專業律師。」

# 輸入資料：法律文件（包含多個搜尋結果）
INPUT_DATA = f"""Here is some research that's been compiled. Use it to answer a legal question from the user.
<legal_research>{LEGAL_RESEARCH}</legal_research>"""
# 翻譯：「以下是已整理的研究資料，用它來回答使用者的法律問題。」

# 範例：教 Claude 如何引用來源
EXAMPLES = """When citing the legal research in your answer, please use brackets containing the search index ID, followed by a period.
<examples>
<example>
The statute of limitations expires after 10 years for crimes like this. [3].
</example>
</examples>"""
# 翻譯：「引用時請用方括號加上搜尋編號，例如 [3]。」

# 任務描述
TASK_DESCRIPTION = f"""Write a clear, concise answer to this question:
<question>{QUESTION}</question>
It should be no more than a couple of paragraphs. If there is not sufficient information, you may say "Sorry, I do not have sufficient information at hand to answer this question." """
# 翻譯：「寫一個簡潔的回答...如果資訊不足，可以說抱歉。」

# 逐步思考
PRECOGNITION = "Before you answer, pull out the most relevant quotes from the research in <relevant_quotes> tags."
# 翻譯：「回答前，先在 <relevant_quotes> 標籤中摘出最相關的引文。」

# 輸出格式
OUTPUT_FORMATTING = "Put your two-paragraph response in <answer> tags."
# 翻譯：「把兩段式的回答放在 <answer> 標籤裡。」

# 預填
PREFILL = "<relevant_quotes>"
```

> **注意：** 這個範例中元素的順序和前面的職涯顧問不同——輸入資料在任務描述之前。**提示詞工程是科學的試錯過程**，鼓勵你混搭調整，找出最適合你需求的組合。
