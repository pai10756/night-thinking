# 第零章：教程使用指南

## 什麼是這個教程？

這是一個互動式教程，教你如何更有效地跟 AI（人工智慧助手 Claude）溝通。就像學習跟一個非常聰明但需要精確指示的新同事說話一樣。

## 事前準備

這個教程**需要一組 API 金鑰（API Key）才能互動操作**。

> **什麼是 API 金鑰？** 你可以把它想成是一把「通行證」，讓你的程式可以合法地跟 Claude 對話。就像進入大樓需要門禁卡一樣，你的程式需要這把金鑰才能使用 Claude 的服務。

如果你沒有 API 金鑰，可以到 [Anthropic Console](https://console.anthropic.com/) 註冊一組。

## 如何開始

### 第一步：安裝必要的工具

```python
# 安裝 Anthropic 的 Python 套件（就像在手機上安裝一個 App）
%pip install anthropic
```

### 第二步：設定你的 API 金鑰

```python
# 把 "your_api_key_here" 換成你自己的 API 金鑰
API_KEY = "your_api_key_here"

# 設定要使用的 AI 模型名稱（就像選擇要跟哪位客服人員通話）
MODEL_NAME = "claude-3-haiku-20240307"

# 儲存這些設定，讓其他頁面也能使用
%store API_KEY
%store MODEL_NAME
```

### 第三步：認識基本工具函式

以下這段程式碼建立了一個叫 `get_completion` 的「輔助函式」。你可以把它想成一個「傳話筒」——你把問題丟進去，它會幫你傳給 Claude，然後把 Claude 的回答帶回來。

```python
import anthropic  # 引入 Anthropic 套件

# 建立一個跟 Claude 的連線（就像撥通電話）
client = anthropic.Anthropic(api_key=API_KEY)

def get_completion(prompt: str):
    """
    輔助函式：把你的提示詞（prompt）傳給 Claude，取得回應。
    就像一個傳話筒——你說什麼，它就幫你問 Claude，再把答案帶回來。
    """
    message = client.messages.create(
        model=MODEL_NAME,          # 使用哪個 AI 模型
        max_tokens=2000,           # 回應最多幾個字（token，約等於一個中文字或半個英文單字）
        temperature=0.0,           # 回答的隨機程度（0 = 最穩定、最一致的回答）
        messages=[
          {"role": "user", "content": prompt}  # 你的問題
        ]
    )
    return message.content[0].text  # 回傳 Claude 的回答文字
```

### 第四步：試試看！

```python
# 你要問 Claude 的話（這就是「提示詞」）
prompt = "Hello, Claude!"

# 把提示詞送出去，印出 Claude 的回答
print(get_completion(prompt))
```

> **小提醒：** 這個教程使用的是 Claude 3 Haiku 模型，並且把 temperature（溫度）設為 0。溫度越低，Claude 的回答越穩定一致，像是考試時按照標準答案回答；溫度越高，回答越有創意但也越不可預測，像是即興發揮。

# 第一章：基本提示詞結構

> **什麼是提示詞（Prompt）？** 就是你對 AI 說的話、你給 AI 的指令。就像你在 Google 搜尋框裡打的搜尋字一樣，但可以更長、更複雜。提示詞的品質直接決定了 AI 回答的品質。

## 課程內容

### Messages API 的基本規則

> **什麼是 API？** API（應用程式介面）就像餐廳的服務生——你不需要走進廚房自己做菜，只要跟服務生（API）點餐，廚房（Claude）就會把菜做好端給你。
>
> **什麼是 Messages API？** 這是 Anthropic 提供的一種「對話格式」，讓你能用標準化的方式跟 Claude 聊天。

要跟 Claude 對話，最少需要提供以下幾個參數（設定值）：

1. **`model`（模型）**：你要使用哪個版本的 Claude。就像選擇你要跟哪位專家諮詢一樣。

2. **`max_tokens`（最大字數上限）**：Claude 最多可以回答多少個 token。
   > **什麼是 Token？** Token 是 AI 處理文字的最小單位。大約等於一個中文字，或是半個英文單字。設定 2000 tokens 大約等於 Claude 最多回答 1000 個中文字左右。注意：這是「硬上限」，Claude 可能會在寫到一半的時候被截斷。

3. **`messages`（訊息）**：你跟 Claude 的對話內容。格式就像即時通訊的聊天記錄，必須標註每一句話是誰說的（`user` 使用者 或 `assistant` 助手）。

另外還有一些可選的參數：

4. **`system`（系統提示詞）**：給 Claude 的「幕後指令」——稍後會詳細說明。

5. **`temperature`（溫度）**：控制回答的隨機程度。0 表示最穩定，數值越高越有創意但也越不可預測。

### 對話的基本格式

想像你在寫劇本，每句台詞都要標明是哪個角色說的：

- **`user`（使用者）**：就是你，問問題的人
- **`assistant`（助手）**：就是 Claude，回答問題的 AI

**重要規則：**
- 對話必須**交替進行**——使用者說一句，助手回一句，使用者再說一句……
- 對話**一定要由使用者開始**
- 不能連續出現兩個使用者的訊息（就像打電話不能兩個人同時講話）

### 範例：正確的提示詞

```python
# 範例 1：簡單的打招呼
PROMPT = "Hi Claude, how are you?"
# Claude 會友善地回覆

# 範例 2：問一個具體問題
PROMPT = "Can you tell me the color of the ocean?"
# Claude 會告訴你海洋的顏色

# 範例 3：問一個事實性問題
PROMPT = "What year was Celine Dion born in?"
# Claude 會回答席琳·狄翁的出生年份
```

### 範例：錯誤的格式

以下是**會出錯**的寫法，幫助你避開常見陷阱：

**錯誤 1：缺少角色和內容欄位**

```python
# 錯誤示範：沒有標明「誰說的」和「說了什麼」
messages=[
  {"Hi Claude, how are you?"}  # 缺少 role 和 content
]
# 這就像寄信沒寫收件人和寄件人，郵局不知道該怎麼處理
```

**錯誤 2：連續兩個使用者訊息**

```python
# 錯誤示範：使用者連說兩句，沒有讓 Claude 回話
messages=[
  {"role": "user", "content": "What year was Celine Dion born in?"},
  {"role": "user", "content": "Also, can you tell me some other facts about her?"}
]
# 這就像你連問兩個問題卻不讓對方回答第一個
```

### 系統提示詞（System Prompt）

> **什麼是系統提示詞？** 想像你在面試一位新員工之前，先在他耳邊悄悄說：「等一下面試的時候，請你用幽默的方式回答所有問題。」這段悄悄話就是系統提示詞——它是你在「正式對話」之前，先給 Claude 的一段「幕後指令」。
>
> 比喻來說：如果一般的提示詞是「考試題目」，那系統提示詞就是「考試規則」。

系統提示詞可以用來：
- 設定 Claude 的行為準則
- 規定回答的風格或格式
- 給 Claude 一個角色設定

**系統提示詞範例：**

```python
# 系統提示詞：告訴 Claude「你的回答規則」
SYSTEM_PROMPT = "Your answer should always be a series of critical thinking questions that further the conversation (do not provide answers to your questions). Do not actually answer the user question."
# 翻譯：你的回答應該總是一系列推動對話的批判性思考問題（不要回答你自己的問題）。不要真的回答使用者的問題。

# 使用者的問題
PROMPT = "Why is the sky blue?"
# 翻譯：為什麼天空是藍色的？

# Claude 不會直接回答，而是會反問一系列思考問題
print(get_completion(PROMPT, SYSTEM_PROMPT))
```

> **為什麼要用系統提示詞？** 一個寫得好的系統提示詞可以大幅提升 Claude 的表現，就像一個好的工作說明書能讓新員工更快上手一樣。

# 第二章：清楚且直接地表達

## 課程內容

### 核心概念：像對新同事說話一樣

**Claude 對於清楚、直接的指令回應最好。**

想像 Claude 是一位剛入職的新同事。除了你告訴他的事情之外，他什麼背景資訊都沒有。你解釋得越清楚、越直接，他的工作成果就越好。

> **清楚提示詞的黃金法則：** 把你的提示詞拿給一個朋友或同事看，請他們按照你的指示去做，看看他們能不能做出你想要的結果。如果他們搞混了，那 Claude 也會搞混。

這就像寫菜譜一樣——如果你寫「加一些鹽」，每個人加的量都不同；但如果你寫「加半茶匙鹽」，結果就會一致得多。

### 範例：寫詩

**情境 1：基本的提示詞**

```python
# 請 Claude 寫一首關於機器人的俳句（一種日本短詩）
PROMPT = "Write a haiku about robots."
# Claude 會寫出俳句，但可能會先說一段「以下是一首俳句……」的開場白
```

這首俳句寫得不錯，但如果你不想要 Claude 在詩前面加一段開場白（例如「以下是一首關於機器人的俳句」）怎麼辦？

**情境 2：更精確的提示詞**

```python
# 加上「跳過開場白，直接寫詩」的指示
PROMPT = "Write a haiku about robots. Skip the preamble; go straight into the poem."
# 翻譯：寫一首關於機器人的俳句。跳過前言，直接進入詩的內容。
# 這次 Claude 會直接給你詩，不會囉唆
```

> **重點：想要什麼，就直接說！** 不要期待 Claude 猜你的心思。你覺得「這不是很明顯嗎？」的事情，對 Claude 來說可能一點都不明顯。

### 範例：要求明確答案

**情境 1：模糊的回答**

```python
# 問 Claude 誰是史上最強的籃球員
PROMPT = "Who is the best basketball player of all time?"
# Claude 可能會列出好幾個名字，然後說「見仁見智」——因為你沒有要求他只選一個
```

**情境 2：強迫 Claude 做選擇**

```python
# 明確要求 Claude 只選一個人
PROMPT = "Who is the best basketball player of all time? Yes, there are differing opinions, but if you absolutely had to pick one player, who would it be?"
# 翻譯：誰是史上最強的籃球員？是的，每個人看法不同，但如果你一定要選一個，你會選誰？
# 這次 Claude 會給出一個明確的答案
```

# 第三章：角色扮演提示法

> **什麼是角色提示法（Role Prompting）？** 就是在對話開始之前，先告訴 Claude「你是某某角色」。就像演員在演戲之前，導演會告訴他「你現在演的是一個海盜船長」，演員就會用海盜的方式說話和行動。

## 課程內容

### 為什麼角色扮演有用？

前面說過，Claude 除了你告訴他的事情之外沒有任何背景資訊。有時候，給 Claude 一個特定的角色身份，可以大幅提升他的表現。

這就像人類一樣——如果有人跟你說「用工程師的思維來想這個問題」，你的思考方式可能真的會不一樣。

角色提示法可以改變：
- Claude 的**寫作風格**（正式 vs. 輕鬆）
- Claude 的**回答角度**（專家 vs. 初學者）
- Claude 的**語氣和態度**（嚴肅 vs. 幽默）
- 甚至可以**提升 Claude 的邏輯和數學能力**！

> **小提醒：** 角色提示可以放在系統提示詞裡，也可以放在一般的使用者訊息裡。

### 範例：觀點大不同

**沒有角色設定：**

```python
# 問 Claude 對滑板的看法
PROMPT = "In one sentence, what do you think about skateboarding?"
# 翻譯：用一句話說說你對滑板的看法？
# Claude 會給一個中規中矩、客觀的回答
```

**加上角色設定——你是一隻貓：**

```python
# 系統提示詞：告訴 Claude 他是一隻貓
SYSTEM_PROMPT = "You are a cat."
# 翻譯：你是一隻貓。

PROMPT = "In one sentence, what do you think about skateboarding?"
# 翻譯：用一句話說說你對滑板的看法？
# 這次 Claude 會用貓的視角來回答——可能會說滑板的輪子很吵、很可怕之類的
```

> **進階技巧：** 你還可以告訴 Claude 他的「觀眾」是誰。「你是一隻貓」和「你是一隻正在對一群滑板手演講的貓」會產生非常不同的回答！

### 範例：角色扮演提升邏輯能力

這是一個經典的邏輯題：

> Jack 看著 Anne。Anne 看著 George。Jack 已婚，George 未婚，我們不知道 Anne 是否已婚。請問：是否有一個已婚的人正在看著一個未婚的人？

**正確答案是「是的」**（因為不管 Anne 已婚還是未婚，結論都成立）。但 Claude 在沒有角色提示的情況下，常常會回答「無法確定」。

**沒有角色設定時：**

```python
PROMPT = "Jack is looking at Anne. Anne is looking at George. Jack is married, George is not, and we don't know if Anne is married. Is a married person looking at an unmarried person?"
# Claude 可能會回答「無法確定」——答錯了！
```

**加上角色設定——你是邏輯機器人：**

```python
# 告訴 Claude 他是一個專門解決邏輯問題的機器人
SYSTEM_PROMPT = "You are a logic bot designed to answer complex logic problems."
# 翻譯：你是一個設計來回答複雜邏輯問題的邏輯機器人。

PROMPT = "Jack is looking at Anne. Anne is looking at George. Jack is married, George is not, and we don't know if Anne is married. Is a married person looking at an unmarried person?"
# 這次 Claude 更可能正確回答「是的」！
```

> **重要觀念：** 在這個課程中，你會學到很多不同的提示詞技巧，它們往往可以達到類似的效果。沒有「唯一正確的方法」，鼓勵你多實驗，找到最適合你的風格！

## 動手練習

### 練習 3.1：數學批改

**背景：** Claude 有時候會在數學方面犯錯。以下是一個解方程式的過程，其中第二步有明顯的算術錯誤：

```
2x - 3 = 9
2x = 6      ← 錯了！正確應該是 2x = 12（兩邊加 3）
x = 3       ← 因為上一步錯了，這裡也錯了
```

**任務：** 修改提示詞或系統提示詞，讓 Claude 判定這個解法是「不正確的」。

```python
# 你可以修改系統提示詞，也可以修改提示詞
SYSTEM_PROMPT = ""

PROMPT = """Is this equation solved correctly below?

2x - 3 = 9
2x = 6
x = 3"""
# 翻譯：以下這個方程式解得對嗎？
```

> **提示：** 試試看用角色提示法！告訴 Claude 他是一個「嚴格的數學老師」或「仔細檢查每一步的數學批改員」。

# 第四章：分離資料與指令

> **什麼是「分離資料與指令」？** 想像你在填一個制式表格：表格的格式（姓名：___、地址：___）是固定的「指令」，而你填入的內容是「資料」。這一章教你怎麼把固定不變的指令和每次都不同的資料分開，讓 Claude 更好地理解你的意思。

## 課程內容

### 為什麼要分離？

在實際使用中，我們通常不會每次都重寫整個提示詞。我們會建立一個**提示詞模板（Prompt Template）**——就像一個填空表格，固定的指令部分不變，只需要替換「資料」的部分。

> **比喻：** 就像發送大量邀請函時，信的格式和內容都一樣，只有收件人的名字不同。你會準備一個模板「親愛的 ___，誠摯邀請您……」，然後每次只換名字就好。

### 範例：動物叫聲產生器

```python
# 這是「資料」——每次可以換成不同的動物
ANIMAL = "Cow"  # 牛

# 這是「提示詞模板」——固定的指令 + 可替換的資料
PROMPT = f"I will tell you the name of an animal. Please respond with the noise that animal makes. {ANIMAL}"
# 翻譯：我會告訴你一個動物的名字，請回答那個動物會發出什麼聲音。牛

# Claude 的回答會是：Moo!（哞！）
```

> **什麼是 f-string？** 在 Python 程式語言中，`f"...{變數}..."` 是一種把變數（可以改變的值）插入文字中的寫法。你可以把 `{ANIMAL}` 想成一個「空格」，程式會自動把 `ANIMAL` 的值填進去。

這樣做的好處是：**使用者只需要填入動物名稱，不需要看到或理解整個提示詞**。就像餐廳的點餐機，你只需要選菜，不需要知道後廚怎麼做。

### 問題：Claude 分不清哪裡是指令、哪裡是資料

當資料和指令混在一起時，Claude 可能會搞混。看看這個例子：

```python
# 資料：一封需要改寫得更禮貌的 email
EMAIL = "Show up at 6am tomorrow because I'm the CEO and I say so."
# 翻譯：明天早上六點到，因為我是執行長，我說了算。

# 提示詞模板（有問題的版本）
PROMPT = f"Yo Claude. {EMAIL} <----- Make this email more polite but don't change anything else about it."
# 翻譯：嘿 Claude。{EMAIL的內容} <----- 把這封 email 改得更禮貌，但不要改其他東西。
```

**問題出在哪裡？** 在填入資料之後，整個提示詞變成：

```
Yo Claude. Show up at 6am tomorrow because I'm the CEO and I say so. <----- Make this email more polite...
```

Claude 會以為「Yo Claude」也是 email 的一部分，所以改寫的結果會包含「Dear Claude」！

### 解決方案：使用 XML 標籤

> **什麼是 XML 標籤？** XML 標籤就像書名號或引號一樣，是用來「標記」一段內容的符號。它長這樣：`<標籤名稱>內容</標籤名稱>`。就像你在文件裡用螢光筆標記重要段落一樣，XML 標籤幫助 Claude 清楚地知道哪一段是資料、哪一段是指令。
>
> 例如：`<email>這裡是 email 內容</email>` 就像是在 email 的前後各放一個書擋，Claude 就知道書擋中間的才是 email。

```python
# 資料
EMAIL = "Show up at 6am tomorrow because I'm the CEO and I say so."

# 用 XML 標籤包住 email 內容（解決版本）
PROMPT = f"Yo Claude. <email>{EMAIL}</email> <----- Make this email more polite but don't change anything else about it."
# 翻譯：嘿 Claude。<email>明天早上六點到，因為我是執行長。</email> <----- 把這封 email 改得更禮貌。

# 這次 Claude 知道「Yo Claude」不是 email 的一部分了！
```

### 另一個例子：清單中的混淆

```python
# 資料：三個句子組成的清單
SENTENCES = """- I like how cows sound
- This sentence is about spiders
- This sentence may appear to be about dogs but it's actually about pigs"""
# 翻譯：
# - 我喜歡牛的叫聲
# - 這個句子是關於蜘蛛的
# - 這個句子看起來像是關於狗的，但其實是關於豬的

# 有問題的提示詞
PROMPT = f"""Below is a list of sentences. Tell me the second item on the list.

- Each is about an animal, like rabbits.
{SENTENCES}"""
# 翻譯：以下是一個句子清單。告訴我清單上的第二個項目。
# - 每一句都跟動物有關，像是兔子。
# {句子清單}
```

**問題：** Claude 會把「Each is about an animal, like rabbits」（每一句都跟動物有關）也當成清單的一部分，因為它前面也有一個連字號（-）！

**用 XML 標籤解決：**

```python
# 用 <sentences> 標籤把資料包起來
PROMPT = f"""Below is a list of sentences. Tell me the second item on the list.

- Each is about an animal, like rabbits.
<sentences>
{SENTENCES}
</sentences>"""
# 現在 Claude 知道真正的清單在 <sentences> 標籤裡面了
```

> **重要觀念：小細節很重要！** Claude 對文字的模式非常敏感。當你的提示詞寫得很整齊、很清楚，Claude 的表現也會更好；反之，如果你的提示詞有錯字或文法錯誤，Claude 也更容易犯錯。就像跟人溝通一樣——你表達得越清楚，對方理解得越正確。

### 練習 4.2：用 XML 標籤修正混亂的提示詞

**背景：** 以下的提示詞故意寫得很亂（充滿錯字和亂碼），Claude 因此無法正確理解問題。

```python

QUESTION = "ar cn brown?"  # 意思是：牠們可以是棕色的嗎？（指狗）

# 翻譯（大意）：嗨是我，我有一個關於狗的問題 [亂碼] 牠們可以是棕色的嗎？[亂碼] 謝謝幫幫忙多多謝快快回答短短謝
```

**任務：** 只加 XML 標籤，不改其他東西，讓 Claude 正確回答「棕色」相關的內容。

> **提示：** 用 `<question>` 和 `</question>` 標籤把真正的問題包起來，這樣 Claude 就能在一堆亂碼中找到真正要回答的問題。

### 練習 4.3：不用 XML 標籤修正

**任務：** 同樣的亂碼提示詞，但這次**不用 XML 標籤**，改為**只刪除一兩個字**來修正。

> **提示：** 想想看，是哪些亂碼字串讓 Claude 最困惑？試著刪掉提示詞中最干擾 Claude 的部分。這個練習讓你看到 Claude 其實能理解相當破碎的語言——問題通常出在某些特別容易混淆的部分。
