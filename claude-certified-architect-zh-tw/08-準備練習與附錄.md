# 準備練習與附錄

---

## 準備練習

完成以下實作練習，幫助你熟悉考試涵蓋的主題。每個練習旨在強化一個或多個考試領域。

### 練習 1：打造一個多工具代理人（含升級邏輯）

**目標：** 練習設計代理人迴圈、工具整合、結構化錯誤處理和升級模式。

**步驟：**
1. 定義 3-4 個 MCP 工具，為每個工具寫詳細的描述（至少兩個工具功能相似，需要靠描述來區分）
2. 實作一個代理人迴圈：檢查 `stop_reason` 來決定繼續執行工具還是結束
3. 為工具加上結構化錯誤回應：包含 `errorCategory`、`isRetryable`、人類可讀描述。測試代理人對每種錯誤的處理（暫時性→重試、商業邏輯→向使用者解釋）
4. 實作一個 Hook 來攔截違反商業規則的工具呼叫（如：超過金額上限→轉人工）
5. 用多重問題的訊息測試，驗證代理人能拆解請求、處理每個問題、綜合出統一回應

**強化的領域：** 領域 1、2、5

---

### 練習 2：為團隊開發工作流程配置 Claude Code

**目標：** 練習配置 CLAUDE.md 層級、自訂斜線命令、路徑規則和 MCP 伺服器整合。

**步驟：**
1. 建立專案級 CLAUDE.md，寫入通用的程式碼標準和測試慣例
2. 在 `.claude/rules/` 建立帶有 glob 模式的規則檔案（如 `paths: ["src/api/**/*"]`），測試規則只在編輯對應檔案時載入
3. 在 `.claude/skills/` 建立帶有 `context: fork` 和 `allowed-tools` 的技能，驗證它在隔離環境中執行
4. 在 `.mcp.json` 配置帶有環境變數的 MCP 伺服器，在 `~/.claude.json` 加入個人實驗用的 MCP 伺服器，驗證兩者同時可用
5. 在不同複雜度的任務上測試計畫模式 vs 直接執行

**強化的領域：** 領域 3、2

---

### 練習 3：打造結構化資料擷取管線

**目標：** 練習設計 JSON Schema、用 tool_use 產出結構化輸出、實作驗證重試迴圈、設計批次處理策略。

**步驟：**
1. 定義一個擷取工具的 JSON Schema：包含必填和選填欄位、enum + "other" 模式、nullable 欄位。測試文件中缺少某些資訊時 AI 是否回傳 null
2. 實作驗證重試迴圈：驗證失敗時，把原始文件 + 失敗結果 + 錯誤訊息一起重送。追蹤哪些錯誤可以重試修復、哪些不行
3. 加入 few-shot 範例展示如何處理不同格式的文件
4. 設計批次處理策略：用 Message Batches API 提交 100 份文件，用 `custom_id` 處理失敗，計算總處理時間
5. 實作人工審查路由：讓模型輸出欄位級信心分數，低信心的轉人工，按文件類型和欄位分析準確率

**強化的領域：** 領域 4、5

---

### 練習 4：設計和除錯多代理人研究管線

**目標：** 練習協調子代理人、管理上下文傳遞、實作錯誤傳遞、處理帶出處的綜合報告。

**步驟：**
1. 建立一個協調者代理人，委派至少兩個子代理人。確保 `allowedTools` 包含 `"Task"`，每個子代理人透過指令（而非自動繼承）接收需要的資訊
2. 實作平行子代理人執行（協調者在一次回應中發出多個 Task 呼叫），測量延遲改善
3. 設計子代理人的結構化輸出：每個發現包含主張、證據摘錄、來源、日期。驗證綜合代理人是否保留了出處
4. 模擬子代理人逾時，驗證協調者收到結構化錯誤資訊，能用部分結果繼續並標註覆蓋缺口
5. 用衝突資料測試（兩個可信來源給出不同數字），驗證綜合報告保留所有數值和出處

**強化的領域：** 領域 1、2、5

## 附錄：技術與概念詞彙表

以下是考試中可能出現的技術和概念，附上白話說明：

### Claude Agent SDK 相關
| 術語 | 白話說明 |
|------|---------|
| Agent Definition（代理人定義） | AI 代理人的「規格表」——描述、指令、可用工具 |
| Agentic Loop（代理人迴圈） | AI 不斷「思考→用工具→看結果→繼續或結束」的循環 |
| `stop_reason` | AI 回應時附帶的「我為什麼停下來」的標記 |
| Hooks | 在特定事件（如工具使用後）自動觸發的程式碼 |
| `PostToolUse` | 工具使用完畢後觸發的 Hook |
| Task 工具 | 用來啟動子代理人的機制 |
| `allowedTools` | 代理人被允許使用的工具清單 |

### Model Context Protocol (MCP) 相關
| 術語 | 白話說明 |
|------|---------|
| MCP Server（MCP 伺服器） | 提供工具和資源的服務，像 USB 集線器一樣連接 Claude 和外部服務 |
| MCP Tools（MCP 工具） | 能「做事」的功能（搜尋、建立、更新、刪除） |
| MCP Resources（MCP 資源） | 能「查看」的資訊（文件目錄、資料庫結構） |
| `isError` | 工具回傳失敗時設定的標記 |
| `.mcp.json` | 專案級的 MCP 配置檔 |
| 環境變數擴展 | 用 `${TOKEN}` 代替真實密碼，避免密碼被上傳 |

### Claude Code 相關
| 術語 | 白話說明 |
|------|---------|
| CLAUDE.md | 給 AI 看的專案說明書 |
| `.claude/rules/` | 分門別類的規則檔案目錄 |
| `.claude/commands/` | 自訂斜線命令的目錄 |
| `.claude/skills/` | 自訂技能的目錄 |
| `context: fork` | 在隔離環境中執行，不汙染主對話 |
| `allowed-tools` | 限制技能可用的工具 |
| `argument-hint` | 提示使用者需要輸入什麼參數 |
| Plan Mode（計畫模式） | 先想清楚再動手 |
| `/compact` | 壓縮上下文以釋放空間 |
| `--resume` | 恢復之前的對話 |
| `fork_session` | 從某個點分出獨立的探索分支 |

### Claude Code CLI 旗標
| 旗標 | 白話說明 |
|------|---------|
| `-p` / `--print` | 非互動模式，執行完就結束（CI/CD 必用） |
| `--output-format json` | 輸出 JSON 格式 |
| `--json-schema` | 指定輸出的 JSON 結構 |

### Claude API 相關
| 術語 | 白話說明 |
|------|---------|
| `tool_use` | 讓 AI 呼叫工具來產出結構化資料 |
| `tool_choice` | 控制 AI 如何選擇工具（自動/必須/強制指定） |
| `stop_reason` | AI 停止的原因（用了工具 or 回合結束） |
| `max_tokens` | 回應的最大長度限制 |
| System Prompt（系統提示詞） | 給 AI 的「背景設定」，使用者看不到 |

### 其他重要概念
| 術語 | 白話說明 |
|------|---------|
| Message Batches API | 批次處理 API，省 50% 但要等最多 24 小時 |
| `custom_id` | 批次中每個請求的識別碼，用來配對結果 |
| JSON Schema | 定義 JSON 資料格式的規範（哪些欄位、什麼類型） |
| Few-shot | 在提示詞中給幾個範例讓 AI 學習 |
| Prompt Chaining（提示鏈） | 把任務拆成固定順序的步驟依序執行 |
| Context Window（上下文視窗） | AI 一次能「記住」的資訊量 |
| Lost in the Middle（中段遺忘效應） | AI 對長文的開頭和結尾記得好，中間容易忽略 |
| Scratchpad（草稿本） | 把關鍵資訊寫到檔案中，需要時重新讀取 |
| Confidence Score（信心分數） | AI 對自己答案的把握程度評分 |
| Stratified Sampling（分層抽樣） | 按類別各取一些樣本來檢查，而非純隨機抽取 |

## 不考的內容

以下主題**不會**出現在考試中：
- 微調 Claude 模型或訓練自訂模型
- API 認證、帳單或帳戶管理
- 特定程式語言或框架的詳細實作
- 部署或託管 MCP 伺服器（基礎設施、網路、容器編排）
- Claude 的內部架構、訓練過程或模型權重
- Constitutional AI、RLHF 等安全訓練方法
- 嵌入模型或向量資料庫
- 電腦操控（瀏覽器自動化、桌面互動）
- 視覺/圖片分析
- Streaming API 或 Server-Sent Events
- 速率限制、配額或定價計算
- OAuth 或 API 金鑰輪換

## 備考建議

1. **用 Claude Agent SDK 打造一個代理人**：實作完整的代理人迴圈、工具呼叫、錯誤處理、會話管理
2. **為真實專案配置 Claude Code**：設定 CLAUDE.md 層級、路徑規則、自訂技能、MCP 伺服器
3. **設計和測試 MCP 工具**：練習寫清楚的工具描述、結構化錯誤回應、測試工具選擇可靠度
4. **打造結構化資料擷取管線**：用 tool_use + JSON Schema、驗證重試迴圈、批次處理
5. **練習提示詞工程技巧**：few-shot 範例、明確審查標準、多輪次審查架構
6. **學習上下文管理模式**：精簡工具輸出、scratchpad 檔案、子代理人委派
7. **複習升級和人機協作模式**：升級時機判斷、信心度路由
8. **完成模擬考試**：在正式考試前先做模擬考（連結會另行提供）
