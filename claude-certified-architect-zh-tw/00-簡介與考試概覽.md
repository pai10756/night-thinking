## 簡介

**Claude 認證架構師 — 基礎級**認證，是用來驗證你是否能在實際工作中，做出合理的技術取捨決策，並使用 Claude 打造真正可上線的應用程式。

### 什麼是「技術取捨」？
> 就像裝潢房子時，你要在「預算」「美觀」「耐用」之間做選擇一樣，打造 AI 應用時也要在「速度」「成本」「準確度」之間做權衡。這個認證就是測你會不會做這些判斷。

這個考試會測試你對以下四大核心技術的基礎知識：

| 技術名稱 | 白話說明 |
|---------|--------|
| **Claude Code** | Anthropic 推出的 AI 程式設計助手，可以直接在命令列（終端機）裡幫你寫程式、改程式、審查程式碼 |
| **Claude Agent SDK** | 一套開發工具包，讓你可以打造「AI 代理人」——也就是能自動執行多步驟任務的 AI 程式 |
| **Claude API** | 應用程式介面，讓你的軟體可以透過網路呼叫 Claude 來處理各種任務（就像用遙控器操作電視一樣，API 就是軟體之間的「遙控器」） |
| **Model Context Protocol (MCP)** | 模型上下文協定，一種標準化的「插頭規格」，讓 Claude 可以連接到外部工具和資料來源（想像成 USB 規格——統一了各種設備的連接方式） |

### 考題取材自哪些真實場景？

- 打造客服聊天機器人（自動處理退貨、帳單爭議等）
- 設計多個 AI 協同工作的研究系統
- 把 Claude Code 整合進軟體開發的自動化流程（CI/CD）
- 打造開發者生產力工具
- 從非結構化文件中自動擷取資料

考生不只要懂概念，還要能對「架構設計」「配置方式」「上線部署的取捨」做出實際判斷。

## 理想考生背景

這個認證的目標考生是**解決方案架構師**（Solution Architect）——也就是負責設計和實作「用 Claude 打造正式上線應用」的人。

> **什麼是解決方案架構師？**
> 就像蓋房子需要建築師來設計整體架構一樣，軟體系統也需要「架構師」來規劃整個系統怎麼搭建、各部分怎麼協作。

理想考生具備以下實務經驗：

1. **用 Claude Agent SDK 打造 AI 代理人**
   - 包含多個 AI 代理人的協作（多代理人協調）
   - 子代理人的任務委派（像主管分配工作給下屬）
   - 工具整合（讓 AI 能呼叫外部服務）
   - 生命週期 Hooks（在特定時機點自動觸發的動作，像是「工具執行完畢後自動做某件事」）

2. **配置和客製化 Claude Code**
   - 使用 CLAUDE.md 檔案來設定團隊工作流程（一個給 Claude 看的「說明書」，告訴它專案的規則和慣例）
   - Agent Skills（代理人技能，可重複使用的指令模板）
   - MCP 伺服器整合
   - Plan Mode（計畫模式，讓 Claude 先想清楚再動手）

3. **設計 MCP 工具和資源介面**
   - 讓 Claude 能連接到後端系統（資料庫、內部 API 等）

4. **設計可靠的提示詞（Prompt）以產出結構化資料**
   - 運用 JSON Schema（一種定義資料格式的規範）
   - Few-shot 範例（給 AI 看幾個示範，讓它學會你要的格式）
   - 資料擷取模式

5. **有效管理上下文視窗**
   > **什麼是上下文視窗？**
   > Claude 一次能「記住」的資訊量是有限的，就像你的桌面大小有限一樣。上下文視窗就是 Claude 的「桌面」，你必須聰明地管理放在上面的東西。
   - 處理長文件、多輪對話、多代理人交接

6. **整合 Claude 到 CI/CD 流程**
   > **什麼是 CI/CD？**
   > Continuous Integration / Continuous Deployment（持續整合/持續部署）。簡單說就是：程式碼一改好，就自動跑測試、自動審查、自動部署上線的一條龍流程。
   - 自動化程式碼審查、測試產生、Pull Request 回饋

7. **做出合理的升級和可靠性決策**
   - 錯誤處理（出錯時該怎麼辦）
   - Human-in-the-loop（人機協作，知道什麼時候該讓人類介入）
   - 自我評估模式（讓 AI 自己檢查自己的答案）

考生通常有 **6 個月以上**使用 Claude API、Agent SDK、Claude Code 和 MCP 的實務經驗。

## 考試內容

### 題型

- 全部都是**單選題**
- 每題有 1 個正確答案和 3 個錯誤選項（干擾項）
- 不答算錯，但不會額外扣分（所以不確定的題目也要猜！）

> **什麼是干擾項？**
> 就是看起來有道理、但其實是錯的選項。專門設計來迷惑「知識不完整」的考生。

### 計分方式

- 通過/不通過制
- 分數範圍：100～1,000 分
- **最低及格分數：720 分**
- 使用「等化計分」（不同版本的考卷難度可能略有不同，分數會經過調整以確保公平）
