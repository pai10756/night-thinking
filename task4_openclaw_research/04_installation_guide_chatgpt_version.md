## 第一次安裝的原則

第一次安裝時，請記住這三件事：

### 1. 先成功，再理解
你現在不用把所有原理都搞懂。  
先裝成功，比先研究細節更重要。

### 2. 一次只做一件事
不要同時研究：
- LINE
- Telegram
- Skills
- 雲端部署
- 自動化腳本

第一次只做「本機安裝成功」就好。

### 3. 只看成功標準
每一章我都會寫：
- 你現在在做什麼
- 成功後應該看到什麼
- 卡住時先檢查什麼

你只要照著做，不要自己加步驟。

## 安裝前準備

### 你需要準備：

- 一台 Windows 10 或 Windows 11 電腦
- 穩定網路
- 可以安裝軟體的權限
- 大約 20～40 分鐘
- 一點耐心

### 建議配備：

- 記憶體 8GB 以上
- 硬碟可用空間 10GB 以上

### 這次會用到的 4 個東西

#### Windows
你現在正在用的系統。

#### WSL2
你可以把它想成：

**在 Windows 裡面開一個 Linux 小工作區。**

OpenClaw 在這種環境通常比較穩。

#### Node.js
OpenClaw 需要的執行環境。  
你先不用理解它的技術原理，只要安裝好就可以。

#### API Key
這是一把讓 OpenClaw 能連到 AI 模型的金鑰。  
沒有它，OpenClaw 不能正常工作。

# 第一章：安裝 WSL2

> 這一步在做什麼：  
> 讓你的 Windows 電腦可以跑 Linux 工具

> 成功後你會得到什麼：  
> 你可以打開一個叫做 Ubuntu 的視窗

---

## 步驟 1：打開系統管理員 PowerShell

請照下面操作：

1. 在 Windows 左下角按右鍵
2. 找到：
   - `Windows Terminal（系統管理員）`
   - 或 `PowerShell（系統管理員）`
3. 點開它
4. 如果系統問你要不要允許，請按 **是**

---

## 步驟 2：輸入安裝指令

把下面這一行完整複製貼上進去：

```powershell
wsl --install
````

貼上後按 `Enter`。

---

## 步驟 3：等待安裝完成

這一步可能需要幾分鐘。
系統會幫你安裝：

* WSL
* 必要元件
* 預設 Linux 發行版（通常是 Ubuntu）

如果畫面要求你重新開機，請照做。

---

## 步驟 4：重新開機

請重新開機。
這一步不要跳過。

---

## 步驟 5：第一次打開 Ubuntu

重新開機後：

1. 打開開始選單
2. 搜尋 `Ubuntu`
3. 點開它

第一次打開時，系統會做初始化。
請稍微等一下。

---

## 步驟 6：設定 Ubuntu 使用者名稱與密碼

Ubuntu 會要你設定：

* 使用者名稱
* 密碼

### 使用者名稱建議這樣填：

* 用英文小寫
* 不要空格
* 不要用中文

例如：

```text
alex
```

### 密碼注意事項

輸入密碼時，畫面**不會顯示字元**，這是正常的。
不是壞掉，也不是沒打進去。

輸入完按 `Enter`，再輸一次確認。

---

## 步驟 7：確認 Ubuntu 有成功打開

如果成功，你會看到類似這樣的畫面：

```bash
yourname@DESKTOP-xxxx:~$
```

只要有看到類似這種格式，就代表成功了。

---

## 步驟 8：檢查 WSL 版本

先把 Ubuntu 關掉。
再回到 PowerShell 或 Windows Terminal，輸入：

```powershell
wsl --version
wsl --list --verbose
```

---

## 成功標準

如果你看到類似下面的結果，就成功了：

```text
NAME      STATE           VERSION
Ubuntu    Running         2
```

最重要的是：

**VERSION 要是 2**

---

## 卡住時先檢查這三件事

### 狀況 1：`wsl --install` 不能用

先確認：

* 你是不是用「系統管理員」打開 PowerShell
* Windows 是否太舊，需要更新

### 狀況 2：Ubuntu 打不開

先重新開機一次再試。

### 狀況 3：Ubuntu 開了但很慢

第一次啟動本來就可能比較久，先等 1～3 分鐘。

# 第二章：安裝 Node.js

> 這一步在做什麼：
> 安裝 OpenClaw 需要的執行環境

> 成功後你會得到什麼：
> 你的電腦可以執行 OpenClaw

---

## 這一章在哪裡操作？

從這一章開始，除非我特別說回 Windows，
不然都在 **Ubuntu 視窗** 裡操作。

也就是：

1. 打開開始選單
2. 搜尋 `Ubuntu`
3. 點進去
4. 在裡面輸入下面的指令

---

## 步驟 1：安裝 nvm

nvm 是幫你安裝 Node.js 的工具。
你先不用記名字，只要照做。

請把下面整段複製貼上到 Ubuntu：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
```

按 `Enter` 後，等它跑完。

---

## 步驟 2：讓設定生效

安裝完後，請輸入：

```bash
source ~/.bashrc
```

---

## 步驟 3：安裝 Node.js

請輸入下面三行：

```bash
nvm install 22
nvm use 22
nvm alias default 22
```

這三行的意思是：

* 安裝 Node.js
* 立即切換到這個版本
* 以後預設都用這個版本

你現在不用理解原理，只要照做。

---

## 步驟 4：確認有沒有裝好

請輸入：

```bash
node -v
npm -v
```

---

## 成功標準

你應該會看到兩行版本號，像這樣：

```text
v22.x.x
10.x.x
```

數字不一定完全一樣，但只要有正常顯示版本號就可以。

---

## 卡住時先檢查這裡

### 狀況 1：`curl` 不能用

請先輸入：

```bash
sudo apt update
sudo apt install -y curl
```

它會叫你輸入密碼。
就是你剛剛在 Ubuntu 設定的那組密碼。

### 狀況 2：`nvm: command not found`

請重新輸入：

```bash
source ~/.bashrc
```

如果還是不行，就把 Ubuntu 關掉重開，再試一次。

### 狀況 3：`node -v` 沒反應

先把 Ubuntu 關掉重開，再輸入：

```bash
nvm use 22
node -v
```

# 第三章：安裝 OpenClaw

> 這一步在做什麼：
> 把 OpenClaw 本體裝到你的電腦裡

## 步驟 1：先安裝必要工具

請在 Ubuntu 輸入：


## 步驟 2：安裝 OpenClaw

接著輸入：

```bash
npm install -g openclaw
```

然後等它跑完。
這一步可能需要幾分鐘。

---

## 步驟 3：確認有沒有裝成功

請輸入：

```bash
openclaw --version
```

---

## 成功標準

如果成功，你通常會看到版本資訊。
只要不是出現「找不到指令」，就代表很可能安裝成功。

---

## 如果它說找不到 `openclaw`

你可以先照下面順序試：

### 方法 1：再裝一次

```bash
npm install -g openclaw
```

### 方法 2：確認 npm 有正常

```bash
npm -v
```

### 方法 3：把 Ubuntu 關掉重開，再試一次

```bash
openclaw --version
```

# 第四章：設定 AI 金鑰

> 這一步在做什麼：
> 讓 OpenClaw 有能力連到 AI 模型

> 成功後你會得到什麼：
> OpenClaw 不再只是空殼，而是真的能回應你

---

## 先用最白話的方式理解

你可以把 OpenClaw 想成一台機器。
但這台機器自己不會思考，它必須連到外部 AI 服務，才有辦法工作。

所以你要準備一把鑰匙，也就是：

**API Key**

---

## 新手建議先用哪一種？

如果你是第一次安裝，建議先用：

**OpenRouter**

理由很簡單：

* 一把金鑰就能接多種模型
* 新手比較容易上手
* 後面要換模型也比較方便

---

## 步驟 1：申請 API Key

先到你要使用的 AI 服務平台申請 API Key。
如果你用 OpenRouter，通常拿到的 key 會長得像：

```text
sk-or-xxxxxxxxxxxxxxxx
```

請先把這串 key 複製好。

---

## 步驟 2：建立設定資料夾與設定檔

回到 Ubuntu，輸入：

```bash
mkdir -p ~/.openclaw
nano ~/.openclaw/.env
```

這兩行的意思是：

* 建立 OpenClaw 的設定資料夾
* 打開設定檔

---

## 步驟 3：把下面三行貼進去

請先貼這個基本版：

```env
OPENROUTER_API_KEY=貼上你的金鑰
OPENCLAW_MODEL_PROVIDER=openrouter
OPENCLAW_MODEL=請填你教學當下指定的模型名稱
```

### 你真正要改的只有右邊

例如：

```env
OPENROUTER_API_KEY=sk-or-xxxxxxxxxxxxxxxx
OPENCLAW_MODEL_PROVIDER=openrouter
OPENCLAW_MODEL=你的模型名稱
```

---

## 這裡怎麼填最安全？

請記住三件事：

### 1. `=` 左邊不要改

你只能改右邊的內容。

### 2. 不要自己亂加引號

不要寫成這樣：

```env
OPENROUTER_API_KEY="sk-or-xxxx"
```

先不要加引號。

### 3. 模型名稱是版本敏感資訊

模型名稱可能會變動。
所以正式發佈教學時，建議你在手冊或影片旁邊補一句：

> 本章模型名稱請以教學發布當日的驗證值為準。

---

## 步驟 4：存檔離開

如果你用的是 `nano`，請這樣做：

1. 按 `Ctrl + O`
2. 按 `Enter`
3. 按 `Ctrl + X`

這樣就存好了。

* 你有建立 `~/.openclaw/.env`
* 你有把金鑰寫進去並存檔

## 步驟 3：如果它出現互動式設定

## 成功標準

對第一次安裝的人來說，成功標準很簡單：

### 成功

* `openclaw` 執行後沒有立刻報錯
* 畫面顯示正在啟動、載入、等待互動，或已進入主介面

### 失敗

* 一執行就報錯
* 說找不到指令
* 說缺少 API key
* 說模型設定錯誤

---

## 如果失敗，先照這個順序檢查

### 檢查 1：OpenClaw 有沒有裝好

```bash
openclaw --version
```

如果這行不行，先回第三章。

---

### 檢查 2：Node.js 有沒有裝好

```bash
node -v
npm -v
```

如果沒有版本號，先回第二章。

---

### 檢查 3：`.env` 有沒有真的存好

```bash
cat ~/.openclaw/.env
```

看一下是不是有你剛剛設定的內容。

> 注意：這行會把你的 key 顯示出來。
> 如果你在錄影、直播或截圖，請小心不要外流。

---

### 檢查 4：是不是 API key 貼錯了

最常見的問題不是安裝失敗，而是：

* key 少貼一段
* 多貼空格
* 貼錯服務商的 key
* 模型名稱填錯

## 常見問題（新手版）

### Q1：我完全不懂 Linux，真的能裝嗎？

可以。
你不用先懂 Linux，只要照著做就行。

### Q2：為什麼不用直接裝在 Windows？

因為很多這類工具在 Linux 環境比較穩。
WSL2 就是在 Windows 裡提供一個比較穩定的 Linux 環境。

### Q3：我一定要先懂 API 嗎？

不用。
你只要先知道 API Key 是一把金鑰，能讓 OpenClaw 連到 AI 服務就夠了。

### Q4：我可以先不接 LINE / Telegram 嗎？

可以，而且建議先不要接。
第一次先把本機版跑起來最重要。

### Q5：如果我中途卡住怎麼辦？

先不要亂改很多地方。
請照這個順序檢查：

1. WSL2 有沒有成功
2. Node.js 有沒有成功
3. OpenClaw 有沒有成功
4. `.env` 有沒有存對
5. API Key 有沒有貼對
