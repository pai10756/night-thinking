### [8:00-11:00] 步驟 4：LINE 整合（重頭戲！）

**【進度圖卡】** `步驟 4/6 — LINE 整合 ⭐⭐⭐`

**【畫面】** LINE 架構流程圖動畫

**【旁白】**
> 好，來到今天的重頭戲——讓你的 AI 助手出現在 LINE 上！
> 先看一下整體架構。

**【流程圖動畫】**
```
手機 LINE → LINE 伺服器 → ngrok 隧道 → 你的電腦 OpenClaw → AI 模型
   ↑                                                         |
   └──────────────── AI 回覆送回 ────────────────────────────┘
```

**【旁白】**
> 你傳訊息到 LINE，LINE 伺服器會透過 ngrok 隧道轉發到你電腦上的 OpenClaw，
> OpenClaw 呼叫 AI 模型產生回覆，再原路送回你的 LINE。
>
> 整個過程需要三件事：
> 1. LINE Developer 帳號
> 2. HTTPS 隧道（ngrok）
> 3. 設定 Webhook

**【螢幕錄影 — Part A：LINE Developers Console】**

**【旁白】**
> 首先，前往 LINE Developers Console。
> 用你的 LINE 帳號登入。
>
> 建立一個 Provider，名字隨意。
> 然後在這個 Provider 下面，建立一個 Messaging API 的 Channel。
>
> Channel 名稱就是你的 Bot 在 LINE 上會顯示的名字。

**【操作畫面】** 依序點擊 Create Provider → Messaging API → 填入資料 → 建立

**【旁白】**
> 建好之後，到 Messaging API 分頁，
> 拉到最下面，點 Issue 產生 Channel access token。
> 然後到 Basic settings 找 Channel secret。
> 這兩組字串等等要用。

**【螢幕錄影 — Part B：ngrok 隧道】**

**【旁白】**
> 接下來要建立 HTTPS 隧道。LINE 的 Webhook 只接受 HTTPS，
> 所以我們需要用 ngrok 把本機的服務暴露到公網。

**【指令放大顯示】**
```
bash scripts/06_ngrok_tunnel.sh
```

**【螢幕錄影】** 展示腳本執行 → 選擇 ngrok → 輸入 auth token → 隧道啟動 → 顯示 URL

**【旁白】**
> 腳本啟動後會顯示你的公開 HTTPS URL。
> 把這個 URL 記下來，加上 /webhook/line，就是你的 Webhook 地址。

**【螢幕錄影 — Part C：設定 OpenClaw + Webhook】**

**【指令放大顯示】**
```
bash scripts/03_line_setup.sh
```

**【旁白】**
> 執行 LINE 設定腳本，貼上剛才的 Token 和 Secret。
> 腳本會自動寫入設定檔。
>
> 然後回到 LINE Developers Console，
> 在 Messaging API 分頁找到 Webhook settings，
> 把 URL 貼進去，打開 Use webhook 開關。
> 記得點 Verify 確認連線。
>
> 最後，**很重要**——關掉 Auto-reply messages。
> 不然 LINE 的自動回覆會跟 OpenClaw 的回覆打架。

**【螢幕錄影 — Part D：測試！】**

**【畫面】** 電腦 + 手機分割畫面

**【旁白】**
> 來測試一下！確認 OpenClaw 正在運行。
> 拿出手機，掃描 Channel 頁面上的 QR Code，加 Bot 為好友。
> 然後傳一則訊息...

**【手機畫面】** 在 LINE 傳送：「你好，請自我介紹一下」

（等待幾秒）

**【手機畫面】** AI 回覆出現

**【旁白】（語氣興奮）**
> 成功了！看到沒有！AI 在 LINE 上回覆了！
> 從現在開始，你就有一個住在 LINE 裡的 AI 助手了。
